# Idea Thinking - 프로젝트 아이디어 인큐베이터

> **목적**: 프로젝트 개선 아이디어를 체계적으로 관리하고 검증된 아이디어를 실제 구현으로 연결
> **생성일**: 2025-12-31

---

## 구조

```
idea-thinking/
├── README.md          # 이 파일
├── main/              # 검증된 핵심 아이디어 (구현 준비 완료)
│   └── *.md           # 실제 구현할 아이디어
└── sub/               # 브레인스토밍 & 탐색 중인 아이디어
    └── *.md           # 검토 중인 아이디어
```

---

## 워크플로우

```
1. 아이디어 발생 → sub/에 문서 작성
2. 탐색 & 프로토타입 → sub/에서 검증
3. 검증 완료 → main/으로 승격
4. 구현 → 실제 코드 반영 후 아카이브
```

### 아이디어 상태

|------|------|------|

| ✅ **승인** | main/ | 검증 완료, 구현 준비됨 |

## 아이디어 문서 템플릿

```markdown
# [아이디어 제목]

> **상태**: 💭 탐색 / 🔬 검증 / ✅ 승인
> **생성일**: YYYY-MM-DD
> **우선순위**: P0 / P1 / P2 / P3

## 배경
왜 이 아이디어가 필요한가?

## 핵심 아이디어
무엇을 어떻게 개선하는가?

## 예상 효과
- 정량적: 성능 향상, 시간 단축 등
- 정성적: 사용성, 유지보수성 등

## 구현 방안
어떻게 구현할 것인가?

## 리스크 & 대안
잠재적 문제점과 대응 방안

## 검증 결과
프로토타입/테스트 결과 (있을 경우)

## 다음 단계
구체적인 액션 아이템
```
