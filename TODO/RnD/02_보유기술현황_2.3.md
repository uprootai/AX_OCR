## 2. RAG (Retrieval-Augmented Generation) 시스템

### 2-1. 고성능 벡터 검색 시스템

○ FAISS 기반 벡터 검색 시스템 구축 경험
  - 대규모 텍스트 데이터베이스에서 유사 사례를 빠르게 검색하는 **실증 완료**

**실제 적용 사례 (NLP 공수산정 프로젝트):**

○ 데이터 규모
  - 고객의 엑셀 개발 요청 텍스트 **1,000+ 건**
  - 384차원 벡터 공간 임베딩

○ 목적
  - 유사한 과거 개발 사례를 검색하여 공수 산정 및 코드 추천

○ 검증된 성과
  - **빠른 검색 속도** (대규모 데이터베이스에서 실시간 검색)
  - **분류 정확도: 89%** (기존 규칙 기반 대비 크게 향상)
  - **높은 공수 산정 정확도** (실제 공수와 근접)

#### 2-1-1. 핵심 기술

**벡터 데이터베이스:**

○ Facebook AI Similarity Search (FAISS)

**임베딩 모델:**

○ OpenAI text-embedding-3-large

**GraphRAG 구현:**

○ 일반 RAG뿐 아니라 GraphRAG를 구현하여 각 사례간의 관계를 참조
  - 세밀한 공수 산정이 가능하도록 진행한 이력 보유

### 2-2. 향후 적용 계획

**도면 메타데이터 임베딩 및 검색:**

○ 도면 번호, 재질, 부품명 등 임베딩 및 검색 계획

**멀티모달 임베딩:**

○ 텍스트 + 이미지 멀티모달 임베딩
  - VLM 통합 시 적용 예정

### 2-3. 텍스트 분류 및 RAG 기반 사례 검색 경험

○ 텍스트 임베딩과 RAG를 활용한 유사 사례 검색 및 견적 산정 시스템 구축 경험 보유

**실제 적용 사례 (NLP 공수산정 프로젝트):**

○ 입력
  - 고객의 엑셀 개발 요구사항 텍스트
  - 예: "판매 데이터를 분석해서 월별 그래프 생성"

○ 처리 과정
  1) 요구사항 텍스트 임베딩 (OpenAI text-embedding-3-large)
  2) FAISS + GraphRAG로 유사한 과거 개발 사례 검색
  3) 과거 사례의 공수 데이터 참조
  4) GPT-4 기반 공수 산정 및 코드 추천

○ 성과
  - 높은 정확도로 공수 자동 산정 달성

**도면 분석 과제 적용 방향 (견적 자동화):**

○ 도면 메타데이터 임베딩 및 유사 도면 검색
  1) 도면 OCR 결과 추출 (치수, GD&T, 재질, 부품명, 도면 번호)
  2) 도면 특징을 텍스트로 구조화
     - 예: "SUS304, 외경 200mm, 평면도 공차 0.05"
  3) 텍스트 임베딩 후 FAISS 벡터 DB에 저장
  4) 신규 도면 입력 시 RAG로 유사한 과거 도면 검색
  5) 과거 도면의 견적 데이터 참조
     - 가공비, 재료비, 공수, 납기
  6) LLM 기반 자동 견적 생성 및 자연어 설명 추가

○ NLP 경험의 직접 활용
  - 요구사항 텍스트 → 공수 산정 (NLP 프로젝트)
  - 도면 메타데이터 → 견적 산정 (도면 분석 프로젝트)
  - 동일한 RAG + LLM 파이프라인 활용 가능
