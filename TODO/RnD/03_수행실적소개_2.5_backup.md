# 2.5 수행 실적 소개

## 사례 1: AI 기반 도면 BOM 자동 추출 시스템 (DrawingBOMExtractor)

### 1-1. 프로젝트 개요

#### 1-1-1. 과제명

**AI 기반 도면 BOM 자동 추출 및 견적 생성 시스템**

#### 1-1-2. 프로젝트 기간

2023년 ~ 현재 (운영 중)

#### 1-1-3. 발주처/협력사

파나시아 (Panasia) - 전기/기계 설비 전문 기업

#### 1-1-4. 프로젝트 목표

CAD/PDF 도면에서 산업 부품을 자동으로 검출하고, 가격 정보가 포함된 BOM(자재명세서)을 자동 생성하여 **견적 작성 시간을 대폭 단축**하는 AI 솔루션 개발

---

### 1-2. 기술적 과제 및 해결 방안

#### 1-2-1. 과제: 다양한 산업 부품의 정확한 검출

**문제점**:

○ 전기 도면에는 27개 이상의 다양한 부품 심볼이 존재
○ 부품 간 형태가 유사하여 오인식 가능성 높음
○ 도면 품질, 스캔 상태에 따라 검출 정확도 차이 발생

**해결 방안**:

○ **YOLOv11/v8 앙상블 시스템** 구축
  - 다중 YOLO 모델의 검출 결과를 투표(Voting) 방식으로 통합
  - 모델별 신뢰도를 가중치로 적용한 Weighted 앙상블
  - 목표 대비 높은 검출 정확도 달성

**기술 스택**:

○ 객체 검출: YOLOv11, YOLOv8
○ 프레임워크: PyTorch, Ultralytics
○ 가속: CUDA (GPU 지원)

#### 1-2-2. 과제: 실시간 처리 성능 확보

**문제점**:

○ 대형 도면 (A0, A1 사이즈)의 높은 해상도 (4K+)
○ 처리 시간이 길어질 경우 사용자 경험 저하

**해결 방안**:

○ **GPU 가속 및 배치 처리**
  - CUDA 활용으로 처리 속도 향상
  - 다중 페이지 PDF 병렬 처리
  - GPU 환경에서 빠른 처리 속도 달성, CPU 환경에서도 안정적 처리

○ **모델 최적화**
  - 경량화된 YOLO 백본 사용
  - 불필요한 레이어 제거 (Pruning)
  - Mixed Precision Training 적용

#### 1-2-3. 과제: 사용자 검증 및 수정 기능

**문제점**:

○ AI 검출 결과 검증 및 보완 필요
○ 사용자가 검출 결과를 검증하고 수정할 방법 필요

**해결 방안**:

○ **대화형 웹 UI 개발** (Streamlit 기반)
  - ✅ 승인 / ❌ 거부 버튼으로 검출 결과 검증
  - ✏️ 클래스 수정 기능 (잘못 분류된 부품 재분류)
  - 🎨 수작업 라벨링 기능 (누락된 부품 직접 추가)
  - 실시간 시각화 (바운딩 박스, 신뢰도 표시)

**[이미지 플레이스홀더]**
*설명: DrawingBOM 웹 UI 메인 화면*
*내용: 도면 업로드 → YOLO 검출 결과 → 검증 버튼 → BOM 생성 화면*

#### 1-2-4. 과제: 다양한 도면 형식 지원

**문제점**:

○ 고객사마다 사용하는 도면 형식 상이 (PDF, JPG, PNG, CAD 등)
○ PDF의 경우 다중 페이지 처리 필요

**해결 방안**:

○ **다중 형식 입력 파이프라인**
  - PDF → 고해상도 이미지 변환 (pdf2image 라이브러리)
  - 페이지 자동 분할 및 개별 처리
  - PaddleOCR 통합으로 텍스트 정보 추가 추출

○ **전처리 최적화**
  - 노이즈 제거 (Gaussian Blur, Bilateral Filter)
  - 대비 향상 (CLAHE)
  - 크기 정규화 (최적 해상도 리사이징)

## 사례 2: NLP 기반 공수산정 및 VBA 코드 추천 시스템

### 2-1. 프로젝트 개요

#### 2-1-1. 과제명

**NLP 기반 VBA 코드 추천 시스템 및 공수산정 모델 개발**

#### 2-1-2. 프로젝트 기간

2024년 ~ 현재 (개발 중)

#### 2-1-3. 프로젝트 목표

고객의 요구사항을 자연어로 입력하면, **AI가 자동으로 카테고리를 분류**하고 **핵심 키워드를 추출**하여 **공수를 산정**하고 **VBA 코드를 추천**하는 시스템 개발

#### 2-1-4. 과제 연관성 (도면 분석 견적 생성)

본 프로젝트는 **자연어 기반 요구사항 분석 → LLM 기반 분류 → 공수/비용 산정**의 전체 파이프라인을 구축한 경험으로, **도면 분석 견적 생성 과제**와 직접적으로 연결됨.

**핵심 유사점**:
○ **입력**: 자연어 요구사항 (NLP 프로젝트) ↔ 도면 이미지 + 메타데이터 (도면 분석)
○ **분석**: 카테고리 분류 + 키워드 추출 ↔ 치수/GD&T 추출 + 특징 분석
○ **산출**: 공수 산정 + 코드 추천 ↔ **견적 생성 + 제조 비용 산정**

**LLM 활용 경험 (도면 분석에 직접 적용 가능)**:
○ ✅ RAG 시스템: 유사 사례 검색 (1,000+ 건 학습 데이터)
○ ✅ GPT-4 기반 의미 이해: 모호한 요구사항 해석
○ ✅ 자동 평가 시스템: 정밀도/재현율 자동 계산
○ ✅ 구조화된 출력: JSON 형식 결과 생성

### 2-2. 기술적 과제 및 해결 방안

#### 2-2-1. 과제: 요구사항 분류 정확도 향상

**문제점**:

○ 고객 요구사항이 자연어로 작성되어 모호함
○ 유사한 카테고리 간 혼동 가능성 높음 (예: "판매·영업" vs "경리·회계")

**해결 방안**:

○ **다단계 임베딩 분류 시스템** 개발
  - 1단계: 대분류 그룹 결정 (관리/행정, 영업/마케팅 등)
  - 2단계: 세부 카테고리 분류
  - 3단계: 특수 케이스 검증
  - 목표를 초과하는 우수한 분류 정확도 달성 ✅

**기술 스택**:

○ 임베딩: OpenAI text-embedding-3-large
○ 벡터 검색: FAISS
○ 리랭킹: BGE Reranker (SentenceTransformers CrossEncoder)
○ LLM: GPT-4o-mini, GPT-4

*내용: 3단계 분류 프로세스 다이어그램*

#### 2-2-2. 과제: 핵심 키워드 추출 성능

**문제점**:

○ 요구사항에서 의미 있는 키워드만 추출 필요
○ 불필요한 단어 제거 및 동의어 처리 필요

**해결 방안**:

○ **RAG + 리랭킹 + LLM 파이프라인**
  1) 유사 사례 검색 (FAISS)
  2) 검색 결과 재정렬 (BGE Reranker)
  3) LLM 기반 키워드 추출 (GPT)
  4) 빈도 및 관련성 기반 순위화

○ **성능 지표**:
  - 정밀도(Precision): 목표 달성 ✅
  - 재현율(Recall): 목표 달성 ✅
  - F1 Score: 목표 달성 ✅

*내용: 입력 요구사항 → 추출 키워드 목록 → 신뢰도 점수*

#### 2-2-3. 과제: RAG 시스템 검색 속도

**문제점**:

○ 1,000+ 건의 학습 데이터에서 유사 사례 검색 시 시간 소요
○ 실시간 응답 필요 (< 1초)

**해결 방안**:

○ **FAISS 벡터 인덱스** 활용

  - 유사도 임계값 최적화
  - 실용적 검색 속도 달성 ✅

○ **캐싱 메커니즘**
  - 임베딩 결과 캐싱

### 2-7. 🎯 도면 분석 견적 생성 활용 방안 (과제 직접 연계)

#### 2-7-1. LLM 기반 공수산정 경험이 도면 견적 생성에 미치는 영향

**본 프로젝트에서 구축한 LLM 활용 노하우는 도면 분석 견적 생성 과제에 직접 적용 가능함.**

| 공수산정 프로젝트 기술 | 도면 견적 생성 적용 방안 | 예상 효과 |
|---------------------|---------------------|---------|
| **RAG 시스템 (FAISS + BGE)** | 유사 도면 검색 및 과거 견적 사례 참조 | 견적 정확도 향상 |
| **GPT-4 기반 의미 분석** | 도면 메타데이터 해석 및 누락 정보 추론 | 자동화율 향상 |
| **다단계 분류 시스템** | 부품 카테고리 자동 분류 (기계/전기/제어) | 높은 분류 정확도 |
| **자동 평가 파이프라인** | 견적 결과 검증 및 품질 관리 | 오류율 감소 |
| **구조화된 출력** | JSON 기반 견적서 생성 및 ERP 연동 | 처리 시간 단축 |
