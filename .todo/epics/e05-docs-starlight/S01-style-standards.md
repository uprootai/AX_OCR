# S01: 문서 구조 규칙 표준화

> **Phase**: 1 (즉시 시작 가능, 프레임워크 무관)
> **예상 소요**: 1~2일
> **의존성**: 없음

---

## 목표

모든 docs-site 페이지에 적용할 **문서 작성 표준**을 정의하고, 기존 문서를 단계적으로 표준에 맞춘다.

## 표준 규칙 정의

### 1. Frontmatter 표준

```yaml
---
sidebar_position: N
title: 한글 제목
description: 한 줄 요약 (검색/미리보기용)
tags: [분류1, 분류2]
---
```

**tags 분류 체계**:
| 카테고리 | 예시 |
|---------|------|
| 시스템 | `architecture`, `devops`, `docker` |
| 분석 | `yolo`, `ocr`, `vlm`, `pipeline` |
| 검증 | `verification`, `human-in-the-loop` |
| 프론트엔드 | `react`, `blueprintflow`, `bom-ui` |
| 고객 | `dsebearing`, `panasia`, `customer` |
| API | `api`, `gateway`, `rest` |

### 2. 페이지 헤더 표준

```markdown
# 제목

> 이 페이지가 무엇을 설명하는지 한 문장으로 요약
```

- H1 바로 아래에 **blockquote 요약** 필수
- 페이지 의도를 즉시 파악 가능

### 3. 섹션 순서 표준

| 순서 | 섹션 | 필수/선택 |
|------|------|----------|
| 1 | `## 개요` 또는 `## 접속 방법` | 필수 |
| 2 | `## 구조` 또는 `## 탭 구성` | 선택 |
| 3 | `## 상세 설명` | 필수 |
| 4 | `## 설정` 또는 `## 환경변수` | 선택 |
| 5 | `## 관련 API` | 선택 (API 연동 시 필수) |
| 6 | `## 관련 문서` | 필수 |

### 4. 표준 표 형식 (4종)

**Type A: 접속/URL 표**
```markdown
| URL | 권한 | 설명 |
|-----|------|------|
```

**Type B: 라우트 맵 표**
```markdown
| 경로 | 페이지 | 설명 | 가이드 |
|------|--------|------|--------|
```

**Type C: 탭/기능 표**
```markdown
| 탭 | 설명 | 주요 동작 |
|----|------|----------|
```

**Type D: 기능 목록 표**
```markdown
| 기능 | 설명 |
|------|------|
```

### 5. Anchor 기반 복합 문서

하나의 페이지에 여러 기능을 설명할 때:

```markdown
## 데이터 관리 도구 {#data-admin}
...

## 컬렉션 관리 {#collection-admin}
...
```

- 다른 문서에서 `[데이터 관리](/docs/admin-tools#data-admin)` 으로 링크

### 6. 문체 규칙

| 규칙 | 예시 |
|------|------|
| 건조한 운영 문체 | ✅ "21개 서비스로 구성된다" |
| 마케팅 문체 금지 | ❌ "강력한 21개 서비스가 제공됩니다" |
| 숫자 명시 | ✅ "52개 라우트", "6개 데이터 도구" |
| 숫자 생략 | ❌ "다양한 라우트", "여러 도구" |
| 추측 금지 | 코드/라우트/컴포넌트 실제 확인 후 작성 |
