# S06: 라우트 문서 강화

> **상태**: ⬜ Todo
> **선행**: S01 (규칙), S04 (섹션 순서)
> **산출물**: route-map.mdx 재작성 + 6개 페이지 상세 강화

---

## 목표

라우트 맵 문서에 권한 구분을 추가하고,
6개 페이지 상세 문서에 접속 방법 표와 anchor 기반 섹션을 적용한다.

## Tasks

### T01 route-map.mdx 재작성

**현재 구조**:
```
라우트 인벤토리 → 사용자 페이지 표 → 관리자 페이지 표 → 시스템 라우트 표
```

**목표 구조**:
```
## 개요
라우트 수, 레이아웃 구조, 로딩 전략 요약

## 라우트 인벤토리
| 구분 | 수 |
|------|-----|
| 공개 페이지 | N개 |
| 관리자 페이지 | N개 |
| 시스템 라우트 | N개 |
| redirect/wildcard | N개 |

## 사용자 페이지
| 경로 | 권한 | 페이지 | 설명 | 가이드 |
|------|------|--------|------|--------|
| `/` | 공개 | Dashboard | ... | [상세](./pages/dashboard) |

## 관리자 페이지
| 경로 | 권한 | 페이지 | 설명 | 가이드 |
|------|------|--------|------|--------|
| `/admin` | 관리자 | Admin | ... | [상세](./pages/admin) |

## 시스템 라우트
| 경로 | 유형 | 설명 |
|------|------|------|
| `*` | wildcard | NotFound 페이지 |

## 라우트 구조
(중첩 라우트, Layout, 지연 로딩 설명)

## 관련 문서
```

**핵심 변경점**:
1. `권한` 컬럼 추가 (공개 / 사용자 / 관리자)
2. redirect, wildcard, wrapper route 별도 섹션
3. 각 행 마지막에 `[상세](...)` 링크
4. 실제 라우트 선언과 표 행 수 1:1 매칭

### T02 6개 페이지 상세 문서 강화

**대상**: `frontend/pages/` 하위 6개 파일

| 파일 | 주요 변경 |
|------|----------|
| dashboard.mdx | `## 접속 방법` 표 추가, 위젯별 anchor |
| project.mdx | `## 접속 방법` 표 추가, 프로젝트 목록/상세 anchor |
| blueprintflow.mdx | `## 접속 방법` 표 추가, 편집기/실행 anchor |
| session.mdx | `## 접속 방법` 표 추가, 세션 목록/상세 anchor |
| admin.mdx | 이미 양호 — anchor 보강만 |
| pages/index.mdx | 하위 페이지 표에 `권한` 컬럼 추가 |

각 파일에 추가할 `## 접속 방법` 표:
```markdown
## 접속 방법

| URL | 권한 | 설명 |
|-----|------|------|
| `/dashboard` | 공개 | 대시보드 메인 |
```

### T03 정합성 검증

- `route-map.mdx` 표 행 수 = 실제 Router 선언 수
- 소스 파일: `web-ui/src/App.tsx` (React Router 선언)
- 각 `[상세](...)` 링크가 실제 파일로 연결되는지 확인

```bash
# 실제 라우트 수 추출
grep -c "path:" web-ui/src/App.tsx

# 문서 표 행 수
grep -c "| \`/" docs-site-starlight/src/content/docs/frontend/route-map.mdx
```
