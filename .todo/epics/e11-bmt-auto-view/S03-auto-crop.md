# S03: 뷰 라벨 + 경계선 조합 자동 분리

> S01(라벨 OCR) + S02(경계선 감지) 조합으로 크롭 좌표를 완전 자동 생성

| 항목 | 값 |
|------|---|
| 상태 | ✅ Done |
| 의존성 | S01 + S02 |
| 핵심 제약 | **하드코딩 좌표 금지** — 모든 좌표는 이미지에서 동적으로 추출 |

---

## 파이프라인

```
이미지 입력
  → PaddleOCR 전체 실행 (뷰 라벨 + TAG 동시)
    → "FRONT VIEW" bbox, "TOP VIEW" bbox 등 추출
      → OpenCV로 각 라벨 주변 경계선 탐색
        → 뷰별 크롭 좌표 자동 생성
          → 각 크롭에 PaddleOCR 재실행 → TAG 추출
```

---

## 검증 방법

E10 S02 수동 크롭 결과(95.8%)와 자동 크롭 결과 비교:
- TAG Recall이 90%+ 유지되면 성공
- 수동 대비 ±5%p 이내면 허용

---

- [ ] 하드코딩 좌표 없이 뷰 분리 동작
- [ ] TAG Recall 90%+ (E10 수동 95.8% 대비)
- [ ] 다른 GVU 도면에도 적용 가능한 구조 (범용성)
