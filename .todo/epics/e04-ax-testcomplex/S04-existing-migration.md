# S04: 기존 3개사 통합 관리

> **목표**: 파나시아/동서기연/테크로스를 E04 범용 체계 안으로 일원화
> **우선순위**: 중간 (S01~S03 완료 후 진행)
> **의존**: S01, S02, S03

---

## 현재 상태 (As-Is)

3개사가 각각 다른 방식으로 관리되고 있음:

| 기업 | 도면 유형 | 진행 상태 | 관리 위치 | 납품 패키지 |
|------|-----------|-----------|-----------|-------------|
| 파나시아 | P&ID (27클래스) | POC 완료, 납품 대기 | E01/E03과 혼재 | 미생성 |
| 동서기연 | 기계 도면 (치수 OCR) | 94도면 분석 완료, 2차 미팅 | E03 전용 Epic | ✅ 완성 |
| 테크로스 | P&ID (60항목) | 요구분석 완료, 개발 착수 예정 | E02 (완료) | 미생성 |

**문제점**:
- 파나시아는 납품 패키지가 아직 없음
- 테크로스는 요구분석만 완료, 실제 개발 미착수
- 3개사의 프로젝트 데이터/회의록이 각각 다른 곳에 분산

## 목표 상태 (To-Be)

```
apply-company/
├── panasia/           ← 기존 + 납품 패키지 생성
├── dsebearing/        ← 기존 유지, 범용 체계로 전환
├── techcross/         ← 기존 + 개발 착수
├── company-04/        ← S05에서 추가
├── ...
└── _template/         ← 고객 디렉토리 템플릿
```

각 고객 디렉토리 표준 구조:
```
{customer}/
├── README.md          ← 고객 개요 (업종, 도면 유형, 담당자)
├── meetings/          ← 회의록
├── requirements/      ← 요구사항
├── delivery/          ← 납품 패키지 (S01 템플릿으로 생성)
└── training/          ← 교육 자료 (S03 산출물 커스터마이징)
```

## 작업 항목

| # | 작업 | 대상 | 산출물 |
|---|------|------|--------|
| 1 | 파나시아 납품 패키지 생성 | 파나시아 | S01 템플릿으로 패키지 |
| 2 | 파나시아 교육 자료 커스터마이징 | 파나시아 | P&ID 특화 퀵카드 |
| 3 | 테크로스 개발 착수 (P&ID 파이프라인) | 테크로스 | S02 절차 적용 |
| 4 | 동서기연 패키지를 범용 구조로 전환 | 동서기연 | 기존 패키지 마이그레이션 |
| 5 | 고객 디렉토리 표준 구조 적용 (3개사) | 전체 | `apply-company/` 통일 |
| 6 | 고객별 상태 대시보드 (진행률 추적) | 전체 | 관리 문서 또는 웹 UI |

## 수용 기준

- [ ] 3개사 모두 `apply-company/{customer}/` 표준 구조로 통일
- [ ] 파나시아 납품 패키지가 `generate-package.sh`로 생성됨
- [ ] 테크로스 개발이 S02 커스터마이징 절차에 따라 착수됨
- [ ] 3개사의 진행 상태를 한 곳에서 추적 가능
