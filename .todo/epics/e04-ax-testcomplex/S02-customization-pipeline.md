# S02: 커스터마이징 파이프라인 표준화

> **목표**: 신규 업종 도면이 올 때 YOLO 클래스 추가 학습 + 전용 파서 생성을 표준 절차로 정립
> **우선순위**: 높음

---

## 현재 상태 (As-Is)

동서기연 커스터마이징이 성공적으로 완료되었으나, 그 과정이 암묵지로만 존재:
- `dsebearing_parser.py` — 수작업으로 생성
- `dsebearing_router.py` — 수작업으로 라우터 3파일 분리
- `dsebearing.yaml` — 수작업으로 API 스펙 작성
- YOLO 27클래스 (파나시아) — 별도 학습, 절차 미문서화

## 목표 상태 (To-Be)

신규 고객이 오면 **3단계 표준 절차**를 따른다:

### Phase 1: 도면 분석 (1~2주)
- 고객 도면 샘플 수집 (최소 10장)
- 도면 유형 판별 (P&ID / 기계 도면 / 혼합)
- 기존 모델로 테스트 → 커버리지 측정

### Phase 2: 모델 적응 (2~4주)
- **P&ID 계열**: YOLO 클래스 추가 라벨링 + Fine-tuning
- **기계 도면 계열**: eDOCr2 파서 확장 또는 전용 파서 생성
- **신규 유형**: 신규 파서/라우터 스캐폴딩 (`create_api.py` 활용)

### Phase 3: 검증 및 납품 (1~2주)
- GT 라벨 생성 → 정확도 측정 (F1 ≥ 90% 목표)
- E2E 테스트 작성
- 납품 패키지 생성 (S01 템플릿 활용)

## 작업 항목

| # | 작업 | 산출물 |
|---|------|--------|
| 1 | 커스터마이징 3-Phase 가이드 문서화 | `docs-site/docs/customization/` |
| 2 | YOLO Fine-tuning 표준 절차서 | 학습 스크립트 + 문서 |
| 3 | 전용 파서/라우터 스캐폴딩 템플릿 | `create_customer_api.py` |
| 4 | 고객별 도면 유형 판별 자동화 도구 | 분류 스크립트 |
| 5 | 커버리지 측정 리포트 자동 생성 | 리포트 템플릿 |

## 수용 기준

- [ ] 신규 고객 도면 10장으로 Phase 1 완료 시 → 커버리지 리포트 자동 생성
- [ ] YOLO Fine-tuning이 스크립트 1개로 실행 가능 (`train_customer.sh --dataset ./data --classes 5`)
- [ ] 전용 파서 스캐폴딩이 5분 이내 완료 (CLI)
- [ ] 동서기연 사례가 이 절차로 재현 가능함을 검증
