# 에이전트 검증 (Agent Verification)

에이전트 검증 시스템은 자동화 효율성과 사람의 감독 사이에서 균형을 맞추는 3단계 하이브리드 접근 방식을 통해 ML 기반 검출 결과의 품질을 보증합니다.

## 문제

ML 기반 객체 검출 및 OCR 추출은 다양한 신뢰도 수준의 결과를 생성합니다. 높은 신뢰도의 검출은 일반적으로 정확하지만, 경계선 및 낮은 신뢰도의 결과는 검토가 필요합니다. 모든 검출을 수동으로 검증하는 것은 대규모에서 비현실적입니다 -- 단일 기계 도면에서 50개 이상의 검출과 100개 이상의 치수 추출이 발생할 수 있습니다.

## 솔루션: 3단계 하이브리드 검증

시스템은 신뢰도 점수와 문맥 신호를 기반으로 각 검출을 세 가지 검증 레벨 중 하나로 라우팅하여, 속도와 정확도 사이의 최적 균형을 달성합니다.

| 레벨 | 이름 | 신뢰도 범위 | 처리 주체 | 일반적 비중 |
|------|------|------------|-----------|------------|
| **L1** | 자동 승인 (Auto-Approve) | >= 0.9 | 시스템 | 전체 항목의 ~60-70% |
| **L2** | 에이전트 검토 (Agent Review) | 0.7 - 0.9 | Claude Sonnet (Vision) | 전체 항목의 ~20-25% |
| **L3** | 사람 검토 (Human Review) | < 0.7 | 사람 운영자 | 전체 항목의 ~5-15% |

전체적으로 80% 이상의 항목이 사람의 개입 없이 처리되며, AI 에이전트의 경계선 사례 검토를 통해 높은 정확도를 유지합니다.

## 주요 기능

- **우선순위 기반 큐**: 중요 항목(낮은 신뢰도)이 검토를 위해 먼저 표시됩니다
- **능동 학습 (Active Learning)**: 모든 검증 결정(승인/거부/수정)이 모델 재학습을 위해 기록됩니다
- **이중 항목 유형**: 심볼 검출과 치수 추출 모두 지원합니다
- **일괄 작업**: 높은 신뢰도 항목을 한 번의 클릭으로 일괄 승인합니다
- **설정 가능한 임계값**: 배포 환경별로 자동 승인 및 중요 임계값을 조정할 수 있습니다
- **대시보드 분석**: 실시간 검증 진행 상황, 에이전트 대 사람 분포 및 정확도 지표를 제공합니다

## 아키텍처 구성 요소

| 구성 요소 | 설명 |
|-----------|------|
| **능동 학습 서비스 (Active Learning Service)** | 우선순위 큐, 임계값 관리, 재학습 데이터 수집 |
| **검증 라우터 (Verification Router)** | 사람 검증 UI 엔드포인트 (큐, 검증, 일괄 승인) |
| **에이전트 검증 라우터 (Agent Verification Router)** | LLM 에이전트 전용 엔드포인트 (base64 이미지, Playwright 호환) |
| **대시보드 (Dashboard)** | 실시간 통계, 작업 분포, 신뢰도 분석 |
