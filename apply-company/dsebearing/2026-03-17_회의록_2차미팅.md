## 1. 이번 미팅의 핵심 결론

이번 2차 미팅에서 가장 중요하게 정리된 내용은 아래 4가지다.

1. `사양서`와 `품질기준서`는 현재 1차 견적 자동화 범위에서 제외한다.
2. BOM에서 `분홍 ASSY`, `기타 서브 도면`이 아니라 `하얀색 단품 도면`이 주 대상이다.
3. 현재 우리가 생각했던 `모든 숫자 추출`이 1차 목표가 아니다.
4. 1차 목표는 각 단품 도면에서 `내경`, `외경`, `폭` 3가지를 먼저 뽑아 리스트화하는 것이다.

---

## 2. 고객 확인으로 다시 확정된 사항

### 2.1 입력 자료 범위

- `사양서`, `품질기준서`는 현재 단계에서 불필요
- BOM 기준으로 조립 구조는 참고하되, 실제 1차 추출 대상은 `하얀색 바탕의 단품 도면`
- 서브 도면의 세부 치수 전체를 다 보는 방향은 아님

### 2.2 1차 자동화 목표

- 각 도면에서 가장 먼저 봐야 할 것은 `외경`, `내경`, `폭`
- 테이블, 영문 설명, 주변 부가 텍스트는 우선순위가 낮음
- 세부 숫자 전체, 모든 주석, 모든 치수값의 완전 추출은 현재 목표가 아님

### 2.3 현재 시스템 방향 수정 필요

- 기존 내부 가정: `도면 내 모든 숫자를 일단 전부 추출`
- 고객이 원하는 방향: `내경 / 외경 / 폭 3개를 우선 추출`
- 따라서 OCR 리콜 개선이나 전체 숫자 승인 UI보다, 우선순위는 `내경 / 외경 / 폭 분류 정확도`로 이동

---

## 3. 미팅 중 질의응답으로 확인된 해석 기준

### 3.1 외경 / 내경 / 폭 우선

- 외경은 도면상 가장 큰 원형 계열 치수
- 내경은 안쪽 원형 계열 치수
- 폭은 단면상 두께 또는 축 방향 길이에 해당하는 값
- 1차적으로는 이 3개만 구조화해도 의미가 있음

### 3.2 불필요하거나 후순위인 정보

- 도면 외곽 테이블
- 일반 영문 문구
- 세부 숫자 전체
- 모든 치수의 완전한 OCR 수집

### 3.3 다음 단계 후보

고객 설명상 `내경 / 외경 / 폭` 이후에는 아래 항목이 후속 후보로 보인다.

- 구멍 수
- 가공 형태
- 홀 관련 표기 해석
- 기타 가공비 영향 요소

다만 이 부분은 2차 미팅에서 개략 설명 수준이었고, 정량 규칙까지 확정되지는 않았다.

### 3.4 정답 샘플 필요성

- 각 도면에서 어떤 치수를 `내경`, `외경`, `폭`으로 볼지는 전문가 판단이 필요한 경우가 있음
- 따라서 대표 도면 몇 개에 대해 정답 위치를 표시한 샘플을 고객에게 요청하는 것이 합리적임
- 이 샘플은 현재 단계에서 모델 성능 개선보다 더 직접적인 기준 데이터가 될 수 있음

---

## 4. 고객에게 요청한 자료

이번 미팅에서 별도로 요청한 자료는 아래와 같다.

- `SS275` 스펙 자료
- 실제 사용 중인 `견적서 양식`

요청 이유는 아래와 같다.

- `SS275` 스펙 자료: 재질 불일치 및 재질 기준 확인
- 견적서 양식: 추출 치수와 실제 견적 항목 연결 구조 파악

고객은 이 자리에서 가능 여부를 바로 확답하기보다, 후속 메일로 정리해 달라는 취지로 응답했다.

---

## 5. 우리 쪽 시연 내용과 고객 반응

### 5.1 시연 내용

- OCR로 숫자를 읽고
- 신뢰도 기준으로 자동 승인하고
- 누락 값은 수동 바운딩 박스로 추가하는 방식 설명

### 5.2 고객 반응

- 현재 컨셉 자체를 부정한 것은 아님
- 다만 `전체 숫자 추출`보다 `외경/내경/폭 우선`이 먼저라고 방향을 재조정함
- 따라서 현재 UI 기반 전체 검수 흐름은 장기 방향일 수는 있어도, 단기 목표와는 다소 어긋나 있음

---

## 6. 이번 미팅에서 드러난 오해

### 6.1 우리가 잘못 이해했던 부분

- 모든 숫자를 먼저 다 읽어야 한다고 이해했던 점
- 치수 전체 추출이 가장 급하다고 봤던 점
- 현재 파이프라인의 주안점을 OCR 양 확대에 두었던 점

### 6.2 이번 미팅으로 수정된 이해

- 우선순위는 `전체 추출`이 아니라 `핵심 3치수 추출`
- 사람이 바로 견적에 활용할 수 있는 최소 데이터셋이 먼저
- 이후에야 구멍 수, 가공 형태 등으로 넘어갈 수 있음
