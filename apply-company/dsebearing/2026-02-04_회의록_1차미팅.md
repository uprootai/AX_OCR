### 2.2 "소재 및 부품 견적 우선"의 의미 확인 [03:28~03:59]

**질문**: 이전 메일에서 "소재 및 부품 견적 우선으로 정리됐으면 좋겠다"고 하신 뜻이 사양 확인 단계에 해당하는 건지?

**김 대표 답변**:

→ **핵심**: "소재 및 부품 견적"은 사양 확인(4일) 단계에 해당하며, 도면의 큰 치수를 보고 여유치를 더해서 소재 견적을 먼저 산출하는 작업임.

### 2.3 BOM 350번 재질 불일치 해명 [03:59~05:33]

**질문**: 메일 파일명에는 "SS400"이라고 적혀있는데 BOM 350번 행에는 "SS275 OR EQV"로 되어있음. 어떤 게 맞는지?

**김 대표 답변**:
> "SS400하고 유사 재질인데, 현재 요 재질은 SS275 또는 동일 재질로 돼 있는데... 금액대가 비슷해서 그냥 단조강이라 적어놓기는 적어놨는데, **명확한 스펙은 275가 맞습니다.**"

→ **결론**: SS400은 금액대가 비슷한 유사 재질이라 대략적으로 적어둔 것. **정확한 스펙은 SS275가 맞음.** 별도 SS275 스펙 자료를 추가로 전달해주겠다고 함.

### 2.4 BOM 계층 구조 설명 [05:33~10:28] ★ 가장 핵심

윤 팀장이 저널 베어링 도면(TD0062015)을 화면에 띄우고, 실제 사양 확인 작업이 어떻게 진행되는지 질문함.

**김 대표가 설명한 BOM 계층 구조**:

BOM에서 색깔별로 도면 유형이 다름.

#### 색깔별 의미

| 색깔 | 의미 | 견적 작업 |
|------|------|----------|
| **분홍색** | 최종 조립 도면 (Assembly) | 치수 추출 불필요 |
| **파란색** | 중간 조립품 (Sub-assembly) | 치수 추출 불필요 |
| **하얀색** | 단품 도면 (개별 부품) | **이것이 실제 견적 대상** |

#### 구체적 예시: BOM 2번 "T1 BEARING ASSY" (TD0062015)

```
BOM 2번: T1 BEARING ASSY (분홍색 = 최종 조립도)
├── BOM 3번: BEARING RING ASSY (파란색 = 중간 조립품)
│   ├── BOM 4번~17번: 개별 부품들 (하얀색 = 단품)
│   └── BOM 18번: 또 다른 조립품 (파란색)
│       └── BOM 19번~23번: 하위 부품들 (하얀색 = 단품)
├── BOM 24번~44번: 나머지 부품들
└── 총 2번~44번까지 합쳐져야 BOM 2번이 됨
```

**김 대표 원문**:
> "2번이 T1 베아링 아셈블리 고립도라고 생각하시면 되고, 그 밑에 4번부터 시작해서 링부터 44번까지가 합쳐진 내용이 2번 도면인데, 세부적으로 이 **단품으로 전부 다 견적을 다 받아야 되는 상황**이고요."

→ **핵심**: 분홍/파란색 도면은 "어디에 부착되는지 확인하는 도면"이고, **하얀색 단품 도면에서만 치수/재질을 추출**해야 함.

### 2.5 조립도 vs 단품도 구분 [10:28~14:33] ★ 핵심

윤 팀장이 "조립도에서 AI가 치수를 추출할 필요가 있는지" 질문함.

**김 대표 답변**:

| 도면 유형 | 추출 필요성 | 이유 |
|----------|------------|------|
| 조립도 (색깔 있는 것) | **불필요** | 수치가 표현이 덜 되어 있음. 조립 사항만 확인하면 됨 |
| 중간 조립도 (TD0062016 등) | **일부 필요** | 정밀한 수치가 있고, 우측 상단에 하위 BOM이 나와있음 |
| 단품도 (TD0062017 등) | **필수** | 실제 수치/형상/재질 정보가 전부 여기에 있음 |

**김 대표 원문**:
> "색깔 있는 도면들은 수치보다는 **조립 내용을 표현하는 도면**들이고, 그 밑에 나와 있는 도면을 보셔야 여기 있는 수치들로 제품의 사이즈라든지 기타 내용들이 다 확인이 될 수 있거든요."

> "색깔 외에 **하얀색으로 표기돼 있는 도면들을 개개별로** 소재 치수부터 시작해서 단품의 견적이 다 나와야지 합산 견적이 나오거든요."

### 2.7 단품 도면에서 추출할 정보 우선순위 [15:24~18:52] ★ 핵심

윤 팀장이 단품 도면(TD0062017 추정)을 보면서 어떤 정보를 뽑아야 하는지 세부 확인함.

#### 도면 내 영역별 필요 여부

| 도면 영역 | 필요 여부 | 김 대표 설명 |
|----------|----------|-------------|
| **큰 소재 사이즈** (Ø550 등) | **1순위 필수** | "이 소재의 치수가 제일 1번으로 빠져야 됨" |
| 구멍 수, 가공 형태 | 2순위 (나중) | "잔잔한 구멍들이 작업에 따라 가공비가 추가로 들어감" |
| 각도 정보 | 2순위 (나중) | "소재 견적 빠지고 나면 2차로 가공 견적 뺄 때 참고로 봄" |
| 가공 표기 (표면처리 등) | 참고 | "요 부위에 대한 가공을 어떻게 해라는 표기들" |
| 상단 공차 테이블 | **불필요** | "일반적인 공차 표시. 견적상 크게 의미 없음" |
| NOTES | **불필요** | 확인 결과 불필요 |
| ISO VIEW | **불필요** | 확인 결과 불필요 |
| SECTION 상세 | 2순위 (나중) | "한참 뒤에 접근해야 되는 부분" |

#### 견적 산출 2단계 프로세스

**김 대표 원문** (가장 중요한 발언):
> "제가 견적을 내는 방법은 **1차적으로 큰 소재 사이즈를 확인해서 소재 비용을 먼저 빼내고**, 그다음에 이런 잔잔한 구멍들이라든지 이런 것들이 작업이 어떻게 되는가에 따라서 **가공비가 추가로 들어가기 시작**하거든요."

> "일단 **소재 사이즈부터 빼내서 정상적으로 빠지면** 그 이후에 이 구멍들의 개수를 파악해서 구멍당 금액이 어떻게 된다는 둥 이렇게 접근하는 게 맞겠나 싶거든요."

```
1단계 (최우선): 큰 소재 사이즈 확인 → 여유치 추가 → 소재 비용 산출
2단계 (이후):   구멍 수, 가공 형태 파악 → 가공비 추가 산출
```

#### 도면 투상법 관련

도면 하나에 여러 뷰(정면, 측면, 단면)가 있는데 이것들은 삼각법으로 표현된 같은 하나의 도면임.

김 대표: "두 개가 도면 삼각법으로 표기한 한 가지 도면이니까 같은 도면입니다."

## 3. 핵심 결론 요약

### 3.1 견적 자동화 범위 확정

| 구분 | 사용 여부 | 이유 |
|------|----------|------|
| BOM | **사용** | 소재 재질, 도면번호, 부품 목록 기준 |
| 단품 도면 (하얀색) | **사용** | 치수, 재질, 가공 정보 추출 대상 |
| 조립도 (분홍/파란색) | **불필요** | 조립 관계 확인용, 치수 없음 |
| 사양서 | **불필요** | BOM에 재질 다 있음 |
| 품질기준서 | **불필요** | 견적과 무관 |

### 3.2 도면 추출 우선순위

```
1순위: 큰 소재 사이즈 (외경, 내경, 길이 등) → 소재비 산출
2순위: 구멍 수, 가공 형태 → 가공비 산출 (나중)
3순위: 각도, 섹션 상세 → 가공비 정밀화 (나중)
불필요: 공차 테이블, NOTES, ISO VIEW
```

### 3.3 해결된 이슈

| 이슈 | 결론 |
|------|------|
| SS400 vs SS275 불일치 | SS275가 정확한 스펙. SS400은 유사 재질로 대략 적어놓은 것 |
| 사양서/품질기준서 활용 여부 | 견적에 안 씀. BOM으로 충분 |

### 3.4 새로 알게 된 사실

1. BOM에 색깔 구분이 있음 (분홍=최종조립, 파란=중간조립, 하양=단품)
2. 견적은 **단품 기준**으로 각각 뽑고 합산하는 방식
3. 소재비가 1차, 가공비가 2차 (순서 있음)
4. 도면의 여러 뷰는 삼각법으로 표현된 같은 부품
