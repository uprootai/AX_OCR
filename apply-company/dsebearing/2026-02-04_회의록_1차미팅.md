## 1. 회의 배경

업루트가 동서기연으로부터 전달받은 94개 PDF 패키지를 1차 분석한 뒤, AI 팀의 QA 및 도메인 이해를 위해 고객과 직접 확인하는 자리임.

여기서 `94개 PDF`는 모두 같은 성격의 도면을 뜻하는 것이 아니라, 아래처럼 서로 성격이 다른 자료 묶음을 의미함.

| 구분 | 수량 | 비고 |
|------|------|------|
| BOM | 1개 | 부품 목록 기준 문서 |
| 사양서 | 2개 | 1차 견적 자동화 범위에서는 제외 가능 |
| 품질 기준서 | 4개 | 1차 견적 자동화 범위에서는 제외 가능 |
| 부품 도면 | 87개 | 조립도/중간 조립도/단품도 혼재 |

즉, 이번 회의의 핵심은 `94개 전체를 다 봐야 하는가`가 아니라, `이 중 무엇이 실제 견적 자동화 대상인가`를 고객 발화 기준으로 좁히는 데 있음.

### 직접 확인된 사실

- 업루트 측은 회의 초반에 이번 미팅이 `94개 PDF 1차 분석 결과`를 가지고 AI 팀 QA 및 도메인 이해를 위해 진행하는 자리라고 설명함.

- 분류 기준은 `BOM`, `사양서`, `품질 기준서`, `부품 도면`의 4대 분류였고, 부품 도면은 다시 5개 하위 분류로 나누었다고 발화함.

### 작성자 해석

### 미확정 사항

- 회의 초반 발화만으로는 `부품 도면 87개 중 정확히 몇 개가 단품도인지`까지는 확정되지 않음.
- 조립도와 중간 조립도 중 일부를 2차 범위에서 다시 참고할 수 있는지 여부는 이 시점에서는 아직 열려 있었음.

### 2.2 "소재 및 부품 견적 우선"의 의미 확인 [03:28~03:59]

**질문**: 이전 메일에서 "소재 및 부품 견적 우선으로 정리됐으면 좋겠다"고 하신 뜻이 사양 확인 단계에 해당하는 건지?

→ **핵심**: "소재 및 부품 견적"은 사양 확인(4일) 단계에 해당하며, 도면의 큰 치수를 보고 여유치를 더해서 소재 견적을 먼저 산출하는 작업임.

#### 정리

- **직접 확인된 사실**: 고객은 사양 확인 단계에서 큰 치수에 여유치를 붙여 소재 견적을 먼저 받는다고 설명함.
- **작성자 해석**: 1차 자동화의 최우선 산출물은 `가공 상세`가 아니라 `소재 산출용 큰 치수`임.
- **미확정 사항**: 여유치 산정 규칙 자체는 이번 회의에서 수치로 확정되지 않음.

- **미확정 사항**: SS275 스펙 자료 원본은 회의 시점에 아직 수령 전이었음.

### 2.4 BOM 계층 구조 설명 [05:33~10:28] ★ 가장 핵심

윤 팀장이 저널 베어링 도면(TD0062015)을 화면에 띄우고, 실제 사양 확인 작업이 어떻게 진행되는지 질문함.

**김 대표가 설명한 BOM 계층 구조**:

BOM에서 색깔별로 도면 유형이 다름.

#### 색깔별 의미

| 색깔 | 의미 | 견적 작업 |
|------|------|----------|
| **분홍색** | 최종 조립 도면 (Assembly) | 치수 추출 불필요 |
| **파란색** | 중간 조립품 (Sub-assembly) | 치수 추출 불필요 |
| **하얀색** | 단품 도면 (개별 부품) | **이것이 실제 견적 대상** |

#### 구체적 예시: BOM 2번 "T1 BEARING ASSY" (TD0062015)

```
BOM 2번: T1 BEARING ASSY (분홍색 = 최종 조립도)
├── BOM 3번: BEARING RING ASSY (파란색 = 중간 조립품)
│   ├── BOM 4번~17번: 개별 부품들 (하얀색 = 단품)
│   └── BOM 18번: 또 다른 조립품 (파란색)
│       └── BOM 19번~23번: 하위 부품들 (하얀색 = 단품)
├── BOM 24번~44번: 나머지 부품들
└── 총 2번~44번까지 합쳐져야 BOM 2번이 됨
```

**김 대표 원문**:
> "2번이 T1 베아링 아셈블리 고립도라고 생각하시면 되고, 그 밑에 4번부터 시작해서 링부터 44번까지가 합쳐진 내용이 2번 도면인데, 세부적으로 이 **단품으로 전부 다 견적을 다 받아야 되는 상황**이고요."

→ **핵심**: 분홍/파란색 도면은 "어디에 부착되는지 확인하는 도면"이고, **하얀색 단품 도면에서만 치수/재질을 추출**해야 함.

#### 정리

- **직접 확인된 사실**: 고객은 BOM 색 구분에 따라 분홍색은 최종 조립도, 파란색은 중간 조립품, 하얀색은 실제 견적 대상 단품이라고 설명함.
- **작성자 해석**: 견적 자동화의 핵심 단위는 `어셈블리 1건`이 아니라 `하얀색 단품 1건`임.
- **미확정 사항**: 모든 도면에서 색 구분이 동일한 품질로 OCR/분석 가능한지는 이번 회의에서 검증되지 않음.

### 2.5 조립도 vs 단품도 구분 [10:28~14:33] ★ 핵심

윤 팀장이 "조립도에서 AI가 치수를 추출할 필요가 있는지" 질문함.

**김 대표 답변**:

| 도면 유형 | 추출 필요성 | 이유 |
|----------|------------|------|
| 조립도 (색깔 있는 것) | **불필요** | 수치가 표현이 덜 되어 있음. 조립 사항만 확인하면 됨 |
| 중간 조립도 (TD0062016 등) | **일부 필요** | 정밀한 수치가 있고, 우측 상단에 하위 BOM이 나와있음 |
| 단품도 (TD0062017 등) | **필수** | 실제 수치/형상/재질 정보가 전부 여기에 있음 |

**김 대표 원문**:
> "색깔 있는 도면들은 수치보다는 **조립 내용을 표현하는 도면**들이고, 그 밑에 나와 있는 도면을 보셔야 여기 있는 수치들로 제품의 사이즈라든지 기타 내용들이 다 확인이 될 수 있거든요."

> "색깔 외에 **하얀색으로 표기돼 있는 도면들을 개개별로** 소재 치수부터 시작해서 단품의 견적이 다 나와야지 합산 견적이 나오거든요."

#### 정리

- **직접 확인된 사실**: 고객은 조립도보다 하얀색 단품 도면을 봐야 실제 사이즈와 기타 내용을 확인할 수 있다고 설명함.
- **작성자 해석**: 조립도는 `관계 이해용`, 단품도는 `견적 산출용`으로 역할을 분리해야 함.
- **미확정 사항**: 중간 조립도(TD0062016 등)의 `일부 필요` 범위가 어디까지인지 세부 기준은 더 필요함.

### 2.6 사양서/품질기준서 필요 여부 [14:41~15:24]

**질문**: 사양서나 품질 기준서는 견적 작업에 어떻게 활용하는지?

**김 대표 답변**:
> "그거는 실질적으로 **견적 볼 때는 안 봐도 되는 부분**이라서 안 쓰셔도 될 것 같습니다. 사양 같은 경우에는 BOM에 어차피 소재 재질하고 다 나와 있으니까 그거로 정리해서 사용하시면 됩니다."

→ **결론**: 견적 자동화 1차 범위에서 사양서(STMPS00095)와 품질기준서(TD0060732)는 제외해도 됨. 즉, 전달받은 94개 PDF 전체가 1차 자동화 대상은 아니며, 우선순위는 `BOM + 단품 도면`임.

#### 정리

- **직접 확인된 사실**: 고객은 견적 볼 때 사양서와 품질기준서는 안 봐도 된다고 발화함.
- **작성자 해석**: 1차 견적 자동화에서 사양서/품질기준서는 기본 입력에서 제외해도 무방함.
- **미확정 사항**: 2차 이후 고도화 단계에서 품질기준서가 전혀 필요 없는지는 이번 회의만으로 확정할 수 없음.

| 가공 표기 (표면처리 등) | 참고 | "요 부위에 대한 가공을 어떻게 해라는 표기들" |
| 상단 공차 테이블 | **불필요** | "일반적인 공차 표시. 견적상 크게 의미 없음" |
| NOTES | **불필요** | 확인 결과 불필요 |
| ISO VIEW | **불필요** | 확인 결과 불필요 |
| SECTION 상세 | 2순위 (나중) | "한참 뒤에 접근해야 되는 부분" |

#### 견적 산출 2단계 프로세스

**김 대표 원문** (가장 중요한 발언):
> "제가 견적을 내는 방법은 **1차적으로 큰 소재 사이즈를 확인해서 소재 비용을 먼저 빼내고**, 그다음에 이런 잔잔한 구멍들이라든지 이런 것들이 작업이 어떻게 되는가에 따라서 **가공비가 추가로 들어가기 시작**하거든요."

> "일단 **소재 사이즈부터 빼내서 정상적으로 빠지면** 그 이후에 이 구멍들의 개수를 파악해서 구멍당 금액이 어떻게 된다는 둥 이렇게 접근하는 게 맞겠나 싶거든요."

```
1단계 (최우선): 큰 소재 사이즈 확인 → 여유치 추가 → 소재 비용 산출
2단계 (이후):   구멍 수, 가공 형태 파악 → 가공비 추가 산출
```

#### 도면 투상법 관련

도면 하나에 여러 뷰(정면, 측면, 단면)가 있는데 이것들은 삼각법으로 표현된 같은 하나의 도면임.

김 대표: "두 개가 도면 삼각법으로 표기한 한 가지 도면이니까 같은 도면입니다."

#### 정리

- **직접 확인된 사실**: 고객은 큰 소재 치수를 1순위로 보고, 이후 구멍/가공 요소를 2차로 본다고 설명함.
- **작성자 해석**: 추출 파이프라인도 `소재비 기준 필드 우선`으로 설계하는 것이 맞음.
- **미확정 사항**: 구멍 수, 가공 형태, 섹션 상세를 어느 시점부터 자동화 범위에 넣을지는 후속 테스트 결과가 필요함.

## 3. 회의에서 직접 확인된 사실

### 3.1 자료 성격

- 이번 회의는 94개 PDF를 BOM, 사양서, 품질기준서, 부품 도면이 섞인 자료 패키지로 분류해 검토하는 전제로 진행됨.
- 업루트 측의 4대 분류(BOM, 사양서, 품질 기준서, 부품 도면)에 대해 녹음상 고객의 즉시 정정 발화는 확인되지 않음.

### 3.2 1차 견적 자동화 입력 범위

- 고객 설명 기준 1차 견적 자동화의 핵심 입력은 `BOM + 하얀색 단품 도면`임.
- 사양서와 품질기준서는 견적 볼 때는 안 봐도 된다고 발화함.
- 분홍색/파란색 조립도는 조립 관계 이해용이며, 직접 치수 추출 대상은 아님.

### 3.3 견적 산출 순서

- 1차는 큰 소재 사이즈 확인과 여유치 반영을 통한 소재비 산출임.
- 2차는 구멍 수, 가공 형태, 각도, 상세 섹션 등을 이용한 가공비 보정임.

### 3.4 데이터 우선순위

- 큰 소재 사이즈는 1순위 필수 항목임.
- 공차 테이블, NOTES, ISO VIEW는 견적상 우선순위가 낮거나 불필요함.
- 여러 뷰는 같은 도면의 삼각법 표현으로 이해해야 함.

### 3.5 개별 이슈

- BOM 350번 재질 이슈의 정확한 스펙은 SS275임.
- SS400은 유사 재질로 대략 적어둔 표현이라고 고객이 설명함.

## 4. 작성자 해석 및 실무 적용 정리

### 4.1 용어 정정

- `94개 터빈 베어링 도면`이라는 표현은 부정확함.
- 더 정확한 표현은 `94개 PDF 자료 패키지` 또는 `94개 전달 자료`임.
- 그 안에는 BOM, 사양서, 품질기준서, 조립도, 중간 조립도, 단품도가 섞여 있음.

### 4.2 1차 자동화 범위

- 당장 자동화해야 할 대상은 `모든 PDF`가 아니라 `BOM + 단품도` 중심임.
- 특히 단품도에서도 첫 단계는 `소재비 산출용 큰 치수`가 우선이다.
- 따라서 OCR/치수 추출 파이프라인도 정밀 가공 정보보다 큰 외형 치수 중심으로 먼저 검증하는 게 맞다.

### 4.3 문서 체계 정리 기준

- 조립도/중간 조립도/단품도는 같은 "도면"으로만 묶지 말고 역할 기준으로 분리 표기해야 함.
- 사양서와 품질기준서는 `전달 자료`에는 포함되지만 `1차 견적 자동화 입력`에는 포함되지 않는다고 별도 표기하는 것이 맞다.
- 이후 다른 문서에서 `94개 전체 도면`, `94개 터빈 베어링 도면`처럼 쓰면 다시 오해가 생길 가능성이 높다.

## 6. 핵심 결론 요약

### 6.1 견적 자동화 범위 확정

| 구분 | 사용 여부 | 이유 |
|------|----------|------|
| BOM | **사용** | 소재 재질, 도면번호, 부품 목록 기준 |
| 단품 도면 (하얀색) | **사용** | 치수, 재질, 가공 정보 추출 대상 |
| 조립도 (분홍/파란색) | **불필요** | 조립 관계 확인용, 치수 없음 |
| 사양서 | **불필요** | BOM에 재질 다 있음 |
| 품질기준서 | **불필요** | 견적과 무관 |

### 6.1-1 용어 정리

- `94개 PDF`는 고객이 전달한 전체 자료 패키지 수량을 의미함.
- `94개 터빈 베어링 도면`이라고 부르면 부정확함.
- 실제로는 BOM, 사양서, 품질기준서, 부품 도면이 섞여 있음.
- 또한 부품 도면 87개 안에서도 조립도와 단품도가 혼재하며, 고객 설명 기준 1차 견적 자동화의 핵심 대상은 `하얀색 단품 도면`임.

### 6.2 도면 추출 우선순위

```
1순위: 큰 소재 사이즈 (외경, 내경, 길이 등) → 소재비 산출
2순위: 구멍 수, 가공 형태 → 가공비 산출 (나중)
3순위: 각도, 섹션 상세 → 가공비 정밀화 (나중)
불필요: 공차 테이블, NOTES, ISO VIEW
```

### 6.3 해결된 이슈

| 이슈 | 결론 |
|------|------|
| SS400 vs SS275 불일치 | SS275가 정확한 스펙. SS400은 유사 재질로 대략 적어놓은 것 |
| 사양서/품질기준서 활용 여부 | 견적에 안 씀. BOM으로 충분 |

### 6.4 새로 알게 된 사실

1. BOM에 색깔 구분이 있음 (분홍=최종조립, 파란=중간조립, 하양=단품)
2. 견적은 **단품 기준**으로 각각 뽑고 합산하는 방식
3. 소재비가 1차, 가공비가 2차 (순서 있음)
4. 도면의 여러 뷰는 삼각법으로 표현된 같은 부품
5. 전달 자료 94개는 전부 같은 성격의 도면이 아니며, 사양서/품질기준서 등 1차 견적 범위에서 제외 가능한 자료가 포함되어 있음
6. 이번 문서는 녹취 원문 전체를 전사한 문서가 아니라, 고객 발화와 실무 적용 포인트를 기준으로 재구성한 구조화 회의록임
