### 1) 자료 구분 확인

요청하신 구분(피팅&계장밸브, 프로세스밸브, 스키드) 중 전달해주신 자료가 각각 어디에 해당하는지 확인 부탁드립니다.

- FB2FB6-32A-1S-D011-1SDNA-36L-KGS 관련 자료 (도면/BOM) → 어떤 구분에 해당하나요?
- W5XGVU-SN2709 관련 자료 (GVU 도면/파트리스트/BOM) → 스키드 구분이 맞나요?
- 아직 전달되지 않은 구분의 샘플이 있는지도 확인 부탁드립니다.

### 2) 도면 형태 확인

전달해주신 도면 PDF가 세 가지 유형으로 확인됩니다.

- FB2FB6 도면 (135KB): 단일 페이지, 부품번호가 도면에 직접 표기된 형태
- W5XGVU-SN2709-GARRB 도면 (2.7MB): 22페이지, 배치도 + 밸브 상세도 + 계장기기도가 통합된 형태
- 3D도면예_GVU_3D (362KB): 3D 렌더링, TAG 라벨만 표기된 형태

실제 운영 시에도 이 세 가지 형태가 모두 검증 대상인지 확인 부탁드립니다.
추가로, 도면에 부품번호가 표기되지 않은 도면 유형도 존재하는지 확인 부탁드립니다.

### 3) 검증 범위 확인

전달해주신 W5XHEGVU-SN2709-DNV.xlsx (ERP BOM)에는 GVU 하위 품목이 1단계로 등록되어 있고, 각 밸브의 하위 부품(Body, Ball Seat 등)은 별도 BOM으로 존재하는 구조인듯한데, 검증 범위가 아래 중 어디까지인지 확인 부탁드립니다.

- 파트리스트 수준 (TAG, 자품목코드, 수량)
- 개별 밸브 하위 부품까지 (재질, 규격, 수량)
- 둘 다??

### 4) 휴먼 에러 발생 유형

말씀하신 "휴먼 에러 발생 유형" 자료가 이와 별도로 존재하는지 확인 부탁드립니다. (이 부분은 추후 확인 되시면)

### 5) 매칭 방식 확인

말씀해주신 예시를 전달받은 자료 기준으로 이해한 내용입니다. 맞는지 확인 부탁드립니다.

**예1) 스키드 (W5XGVU-SN2709 자료 기준)**

도면 PDF의 TAG → 파트리스트의 CODE/Drawing No. → ERP BOM의 자품목코드 순서로 매칭하는 구조로 이해했습니다.

도면 TAG "V01"
  → 파트리스트(W5XGVU-SN2709-PLRA) CODE "V01" / Drawing No. "FB2F29-R64A-A111-36L-EXT-GVU-DNV"
    → ERP BOM(W5XHEGVU-SN2709-DNV) 자품목 "FB2F29-R64A-A111-36L-EXT-GVU-DNV" / 수량 1EA

**예2) 개별 밸브 (FB2FB6 자료 기준)**

도면 PDF의 부품리스트와 ERP BOM을 직접 비교하는 구조로 이해했습니다.

도면 부품리스트 항목4 "BS-85-6F-T1" / 수량 2EA
  → ERP BOM(FB2FB6 BOM.xlsx) 자품목 "BS-85-6F-T4" / 수량 2EA
    → 불일치: 재질 상이 (T1 vs T4)

이외에 다른 매칭 방식이 적용되는 케이스가 있으면 안내 부탁드립니다.

확인 후 회신 부탁드립니다.
감사합니다.

## 회신 내용 (2026-03-24 수령)

### 1) 자료 구분 → 확인 완료

| 구분 | 정의 | 해당 자료 |
|------|------|-----------|
| **프로세스밸브** (단품) | 단품 밸브 조립도 + BOM | 1_FB2FB6 (도면/BOM) |
| **스키드** (시스템) | Gas Valve Unit 전체 | 2_W5XGVU-SN2709 (GVU 도면/파트리스트/BOM) |
| **피팅&계장밸브** | Instrumentation 도면 | 샘플 미수령 (추가 요청 필요) |

### 2) 도면 형태 → 확인 완료

- 단일 페이지 + 다중 페이지 **모두 검증 대상**
- **부품번호 없는 도면 존재함** (GA / Layout 도면)

### 3) 검증 범위 → **둘 다 (2단계 구조)**

- **1단계**: 파트리스트 수준 검증 (TAG, 자품목코드, 수량)
- **2단계**: 하위 부품 검증 (재질, 규격, 수량)

### 4) 휴먼 에러 유형 → 추후 수령 예정

### 5) 매칭 방식 → 확인 완료 + 보정

- **예1 (스키드)**: 맞음 (TAG → 파트리스트 → ERP BOM)
- **예2 (단품 밸브)**: 보정됨 — 3단계 구조
  - 도면 **Balloon No** → **Part List (부품번호)** → **ERP BOM 코드**
  - 기존 이해(도면 부품리스트 → ERP 직접 비교)와 다름. 중간에 Part List가 별도로 존재
