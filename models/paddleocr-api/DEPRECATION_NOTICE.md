# PaddleOCR API - Upgrade Notice

> **상태**: UPGRADED to 3.0.0 (PP-OCRv5)
> **날짜**: 2025-12-31
> **이유**: SOTA 분석 결과 PaddleOCR 3.0이 13% 정확도 향상 및 106개 언어 지원

---

## 업그레이드 이력

### v3.0.0 (2025-12-31) - PP-OCRv5
- **정확도**: PP-OCRv4 대비 13% 향상
- **언어**: 106개 언어 지원
- **기능**: 세로 텍스트, 필기체 인식 개선
- **구조**: PP-StructureV3 문서 파싱

### v1.0.0 (이전 버전)
- 이전에는 eDOCr2와 기능 중복으로 삭제 권장되었음
- SOTA Gap 분석 결과 PaddleOCR 3.0의 가치가 재평가됨

---

## 현재 상태

**PP-OCRv5 주요 개선사항**:

| 항목 | PP-OCRv4 | PP-OCRv5 | 개선 |
|------|----------|----------|------|
| 정확도 | 기준선 | +13% | 상당 |
| 언어 수 | ~80 | 106 | 26개 추가 |
| 세로 텍스트 | 제한적 | 우수 | 대폭 개선 |
| 필기체 | 제한적 | 개선됨 | 개선 |
| 모델 크기 | ~100MB | ~100MB | 동일 |

---

## eDOCr2와의 역할 분담

| 기능 | PaddleOCR 3.0 | eDOCr2 | 권장 |
|------|---------------|--------|------|
| **일반 텍스트 OCR** | 우수 | 양호 | PaddleOCR |
| **다국어 OCR** | 106개 언어 | 영어 전용 | PaddleOCR |
| **치수 인식** | 제한적 | 우수 (93.75%) | eDOCr2 |
| **GD&T 인식** | 없음 | 우수 (~90%) | eDOCr2 |
| **도면 전문성** | 범용 | 전문화 | eDOCr2 |

**결론**: 두 API는 상호보완적이며, 각각의 역할이 명확함

---

## 사용 사례

### PaddleOCR 3.0 권장
- 다국어 도면 텍스트 인식
- 일반 문서 OCR
- 세로 텍스트가 많은 도면
- 필기 메모 인식

### eDOCr2 권장
- 엔지니어링 치수 추출
- GD&T 기호 인식
- 도면 표 추출
- 영문 엔지니어링 도면

## API 엔드포인트

```bash
# PaddleOCR 3.0 API
curl -X POST http://localhost:5006/api/v1/ocr \
  -F "file=@drawing.png" \
  -F "lang=korean" \
  -F "min_confidence=0.5"

# 헬스 체크
curl http://localhost:5006/api/v1/health
```
