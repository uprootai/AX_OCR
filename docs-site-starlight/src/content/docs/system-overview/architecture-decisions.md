# Architecture Decisions

> eDOCr, VL API, YOLOv11 세 모델의 정량적 비교와 최종 채택 근거를 정리한다.

---

## 빠른 비교표

| 평가 항목 | eDOCr v1/v2 | VL API | **YOLOv11** |
|----------|-------------|--------|-------------|
| **F1 Score** | 8.3% | 70-85% | **96.3%** |
| **월간 비용** | 무료 | $45-120 | **무료** |
| **처리 속도** | 34초 | 45초 | **5-15초** |
| **커스터마이징** | 어려움 | 불가능 | **쉬움** |
| **데이터 보안** | 온프레미스 | 외부 API | **온프레미스** |
| **확장성** | 낮음 | 낮음 | **높음** |
| **총점** | **2/8** | **3/8** | **7/8** |

## 상세 비교

### 1. 정확도 (가중치: 40%)

| 모델 | F1 Score | 평가 |
|------|----------|------|
| eDOCr v1/v2 | 8.3% | 실용 불가능 - 오검출 66%, 주요 치수 44% 누락 |
| VL API (Claude 3.5 Sonnet) | 70-85% | 실용 가능 - Information Block 90-95% |
| **YOLOv11** | **96.3%** | 최고 수준 - GD&T 95%+, OBB 지원 |

### 2. 비용 (가중치: 30%)

| 모델 | 초기 비용 | 월간 운영 | 연간 비용 |
|------|----------|----------|----------|
| eDOCr | 무료 | $0 | $0 (정확도 8.3%로 실용 불가) |
| VL API | 없음 | $45-90 | $540-1,080 |
| **YOLOv11** | **무료 (오픈소스)** | **$0** | **$0** |

### 3. 처리 속도 (가중치: 15%)

| 모델 | 단일 도면 | 100장 배치 |
|------|-----------|------------|
| eDOCr v1 | 34초 | 57분 |
| VL API | 45초 | 75분 |
| **YOLOv11 (CPU)** | **12초** | **20분** |
| **YOLOv11 (GPU)** | **5초** | **8분** |

### 4. 커스터마이징 (가중치: 10%)

- **eDOCr**: CRAFT + CRNN 파이프라인 복잡, 재학습 어려움
- **VL API**: 프롬프트만 수정 가능, 모델 학습 불가
- **YOLOv11**: 자체 데이터로 쉽게 학습 가능

```python
# YOLOv11 자체 데이터 학습 예시
model = YOLO("yolo11n.pt")
model.train(data="custom_drawings.yaml", epochs=100)
```

### 5. 데이터 보안 (가중치: 5%)

| 모델 | 보안 수준 | 비고 |
|------|----------|------|
| eDOCr | 안전 | 온프레미스, 인터넷 불필요 |
| VL API | 위험 | 외부 API, 규정 준수 이슈 |
| **YOLOv11** | **안전** | **온프레미스, 자체 GPU** |

## 최종 권장사항

### 채택: YOLOv11

**근거**:
1. **무료** (월 $45-120 절감)
2. **최고 정확도** (F1 96.3% 달성 가능)
3. **빠른 처리** (5-15초, eDOCr 대비 3-7배)
4. **커스터마이징** (자체 데이터 학습)
5. **데이터 보안** (온프레미스)
6. **확장 가능** (실시간/배치 처리)
7. **2025년 최신 연구로 증명됨** (arXiv 2510.21862)

### 폐기: eDOCr v1/v2

- F1 8.3% → 실용 불가능
- Gateway API에서 엔드포인트 유지 (하위 호환성)
- 기본값을 YOLOv11로 변경

### 조건부 사용: VL API

- YOLOv11 구축 전 임시 솔루션
- 비용 예산 확보된 경우
- 월 처리량 1000장 이하, 비민감 데이터만

## 모델 선정 기준

### 핵심 요구사항

| 항목 | 요구사항 | 우선순위 |
|------|---------|---------|
| 정확도 | Recall > 90%, Precision > 85% | Critical |
| 처리 속도 | < 60초/장 | High |
| 운영 비용 | 월 < $200 | High |
| 데이터 보안 | 온프레미스 처리 | Critical |
| 확장성 | 모델 학습 가능 | High |

### YOLOv11 선택 이유 (7가지)

1. **최고 정확도 (F1 96.3%)** - 기계 도면 분석에 최적화, 실제 연구로 증명
2. **비용 효율성 (무료)** - 초기 학습 무료 (Colab GPU), 월간 $0
3. **처리 속도 (5-12초)** - eDOCr 대비 3-7배 빠름
4. **데이터 보안 (온프레미스)** - GDPR, ISO 27001 규정 준수
5. **커스터마이징 가능** - 자체 도면으로 모델 학습
6. **확장성** - 배치 처리 (100장/20분), 실시간 처리 (5-160 FPS)
7. **최신 기술** - 2025년 최신 모델, OBB 지원

## 실행 계획

### Phase 1: Proof of Concept (Week 1)

```bash
pip install ultralytics
python yolo_prototype.py --source /path/to/drawings
python evaluate.py --ground_truth annotations.json
```

**판단 기준**: F1 > 40% → Phase 2 진행

### Phase 2: MVP (Week 2-4)

```bash
# 데이터 라벨링 (Roboflow) - 100장 → 증강 → 500장
yolo train data=drawings.yaml model=yolo11n.pt epochs=100
docker-compose up -d yolo-api
```

**판단 기준**: F1 > 70% → 배포

### Phase 3: 고도화 (Month 2-3)

```bash
yolo train data=drawings_large.yaml model=yolo11m.pt epochs=200
python batch_inference.py --gpus 0,1,2,3
```

**목표**: F1 > 90%, 처리 속도 < 10초/장
