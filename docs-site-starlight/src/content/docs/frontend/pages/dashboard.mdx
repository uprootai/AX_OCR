# 대시보드 (Dashboard Page)

> API 상태, Docker 컨테이너, 프로젝트 현황, 통계를 한 화면에서 확인하는 통합 대시보드 페이지이다.

## 접속 방법

| URL | 권한 | 설명 |
|-----|------|------|
| `/dashboard` | 공개 | 루트(`/`) 접근 시 자동 리다이렉트 |

<a id="page-structure"></a>

## 페이지 구조

Dashboard 컴포넌트(`pages/dashboard/Dashboard.tsx`)는 7개 섹션으로 구성된다. 상단 네비게이션 바에서 각 섹션으로 스크롤 이동할 수 있다.

| 섹션 ID | 이름 | 컴포넌트 | 설명 |
|---------|------|----------|------|
| `section-api` | API 상태 | `APIStatusMonitor` | 21개 AI 서비스 헬스체크, GPU VRAM, 카테고리별 가동률 |
| `section-containers` | 컨테이너 | `ContainerManager` | Docker 컨테이너 상태 확인 및 시작/중지 |
| `section-projects` | 프로젝트 현황 | 인라인 | 프로젝트 목록, 세션 수, 완료/대기 상태, 진행률 |
| `section-quick-actions` | 빠른 실행 | 인라인 | BlueprintFlow 빌더, 템플릿, 모니터링 바로가기 |
| `section-stats` | 통계 | 인라인 | 오늘 분석 건수, 성공률, 평균 응답 시간, 에러 수 |
| `section-custom-apis` | Custom APIs | 인라인 | 사용자 등록 API 목록 (활성화/비활성화, 내보내기, 삭제) |
| `section-getting-started` | Getting Started | 인라인 | 3단계 시작 가이드 |

<a id="details"></a>

## 상세 설명

### API 자동 검색
앱 최초 실행 시 Gateway의 `/api/v1/registry/list`를 호출하여 등록된 API를 자동으로 검색한다. `localStorage`에 `auto-discovered` 플래그를 저장하여 1회만 실행한다.

### 프로젝트 현황
`projectApi.list()`로 프로젝트 목록을 가져온 뒤, 각 프로젝트의 `projectApi.get()`을 병렬 호출하여 세션 정보를 로드한다. 요약 통계(프로젝트 수, 전체 세션, 완료, 대기)를 그리드로 표시한다.

### Custom API 관리
`apiConfigStore`에서 관리하는 Custom API 목록을 표시한다. 각 API를 활성화/비활성화하거나, Built-in API 코드로 내보내기(`ExportToBuiltinDialog`)하거나, 삭제할 수 있다.

<a id="related-api"></a>

## 관련 API

| 메서드 | 엔드포인트 | 설명 |
|--------|-----------|------|
| GET | `/api/v1/registry/list` | Gateway에 등록된 API 목록 자동 검색 |
| GET | `BOM /api/v1/projects` | 프로젝트 목록 조회 |
| GET | `BOM /api/v1/projects/:id` | 프로젝트 상세 (세션 포함) |
