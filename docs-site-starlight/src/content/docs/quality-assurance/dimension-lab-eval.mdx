## 파이프라인

배치 평가는 세션당 다음 순서로 실행된다:

1. **eDOCr2 OCR** — TensorFlow/Keras 기반 치수 텍스트 인식 (타임아웃 600초)
2. **K방법 전체 파이프라인** — `extract_by_geometry` 직접 호출 (타임아웃 180초)
   - 원 검출 (OpenCV Contour + HoughCircles)
   - 치수선 검출 + ROI 크롭 OCR (PaddleOCR)
   - ID 최소값 제약: `max(30, OD * 0.10)`
   - W 최소값 제약: `OD * 0.05`
   - 스케일 일관성 검증 + 크기 보너스 점수 결정
3. **H방법 분류** — 값 크기 순위로 OD(최대)/ID(중간)/W(최소)
4. **S방법** — 세션명에서 치수 파싱 (예: `T3 (380×260)` → OD=380, ID=260)
5. **최종 선택** — `_pick_best()`: K → H → S 우선순위 폴백

## 주요 발견

### 성공 패턴

스러스트베어링(GT 100%)은 K방법 스케일 일관성 알고리즘이 가장 잘 동작하는 케이스. 외원/내원 px/mm 비율 일관성 검증이 정확한 OD/ID 선택에 기여.

### OD 과대 검출 (T5, T6)

플랜지/볼트원 직경이 실제 베어링 OD보다 큼. 도면 본체에 베어링 OD가 치수 주석이 아닌 타이틀 블록에만 기재된 경우 K방법은 추출 불가. `_pick_best()`의 S방법 폴백으로 대응.

### ID 파싱 오류 (TD0062019)

`(3)22` → `3`으로 파싱됨. 괄호가 참조치수 마커가 아닌 부품번호/주석 번호인 경우 오파싱 발생. 숫자 파서의 괄호 처리 로직 개선 필요.

### T8 Ø기호 문제

Ø1076, Ø1050은 볼트원 직경이나 K방법이 이를 OD/ID로 분류. Ø값 필터링 또는 가중치 조정 필요.

## K방법 알고리즘 변경 이력

### v1 — 스케일 일관성 점수 개선

OD/ID 후보 쌍의 `px/mm` 비율 일관성만으로 점수를 매기면 작은 값 쌍이 큰 값 쌍보다 높은 점수를 받는 회귀 발생. 크기 보너스를 주 점수로 변경.

```python
# 크기 보너스(주) + 스케일 일관성(보조)
magnitude_score = od_c["num"] / max_val  # 최대값 대비 비율
score = magnitude_score * 1.5            # 크기 우선 (최대 +1.5)
score += max(0, 1.0 - consistency * 3) * 0.5  # 일관성 보조 (최대 +0.5)
```

### v2 — ID/W 최소값 제약

작은 치수(챔퍼, 볼트홀)가 ID/W로 분류되는 문제 해결:

```python
# ID 최소값: 절대 30mm 이상 + OD의 10% 이상
id_min = max(30, od_c["num"] * 0.10)

# W 최소값: OD의 5% 이상
w_candidates = [u for u in unique if u["num"] >= od_c["num"] * 0.05]

# 폴백: 30mm 이상만 허용
valid = [u for u in unique if u["num"] >= 30]
```

### v3 — 전체 파이프라인 강제 (이번 배치 적용)

배치 평가에서 K방법 실행 시 `_classify_by_circle_proximity` 숏컷을 제거. 항상 `extract_by_geometry` 전체 파이프라인을 호출하여 ID/W 최소값 제약이 반드시 적용되도록 변경.
