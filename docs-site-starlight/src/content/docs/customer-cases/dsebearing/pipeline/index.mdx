> 기하학 카테고리 4개 방법(K/L/M/N)이 도면에서 OD/ID/W를 어떻게 추출하는지, ASSY 3개 + 단품 1개 = 4개 도면에 적용한 결과를 비교한다.

## 방법론 전체 맥락

Dimension Lab은 **4개 카테고리 × 14개 방법**으로 OD/ID/W를 추출·비교한다.
이 섹션에서는 **기하학 카테고리(K/L/M/N)** 4개 방법을 상세 분석한다.

| 카테고리 | 코드 | 방법 수 | 접근 방식 |
|---------|------|:---:|----------|
| 기호 기반 | A, B, C, D | 4 | OCR 텍스트의 Ø/R 기호·패턴 분석 |
| 통계/위치 | E, F, G | 3 | 치수 크기·위치·파일명 활용 |
| 파이프라인 | H, I, J | 3 | 순차 규칙 적용 (크기순, 휴리스틱) |
| **기하학** | **K, L, M, N** | **4** | **원 검출·치수선 토폴로지 분석** ← 이 섹션 |

## 기하학 4개 방법

### K — 기하학기반 (`geometry_guided`)

4단계 독립 파이프라인으로 OD/ID/W를 추출한다.

| 단계 | 처리 | 알고리즘 |
|:---:|------|----------|
| 1 | 원 검출 | Contour + 타원 피팅 → HoughCircles 폴백 |
| 2 | 치수선 검출 | LSD — 원 근처 선분 필터링 |
| 3 | ROI 크롭 + OCR | Focused(1.8×r) + Wide(3.0×r) 이중 크롭 |
| 4 | 분류 | 스케일 일관성 + 크기 순위 + 원 근접도 |

- **강점**: 원형이 도면 중앙에 크고 동심 구조일 때 높은 정확도
- **약점**: 원 검출 실패 시 전 단계로 오류 전파

### L — 끝점토폴로지 (`endpoint_topology`)

치수선의 양 끝점 위치로 R(반지름)과 Ø(직경)를 구분한다.

| 단계 | 처리 |
|:---:|------|
| 1 | K와 동일한 원 검출 |
| 2 | LSD로 치수선 검출 → 각 끝점의 위치 분석 |
| 3 | 끝점이 원 중심 근처 → R(반지름), 원주 위 → Ø(직경) |
| 4 | R 값은 ×2로 직경 변환 → OD/ID 크기순 배정 |

- **강점**: 치수선 끝점의 기하학적 의미를 직접 해석
- **약점**: 끝점 판정이 원 검출 정확도에 의존. 복잡한 도면에서는 대부분 `unknown` 판정

### M — 심볼검색 (`symbol_search`)

OCR 텍스트에서 R/Ø 기호를 정규식으로 검색한다.

| 단계 | 처리 |
|:---:|------|
| 1 | OCR 결과 전체를 정규식 스캔 |
| 2 | `Ø{숫자}` → 직경(DIAMETER)으로 분류 |
| 3 | `R{숫자}` → 반지름 → ×2로 직경 변환 |
| 4 | 직경 값 중 최대=OD, 차대=ID |

- **강점**: 원 검출과 독립적 — Ø 기호가 명확하면 바로 OD 식별
- **약점**: Ø 접두사 없는 치수(예: `515`)는 탐지 불가

### N — 중심레이캐스트 (`center_raycast`)

검출된 원 중심에서 8방향 레이를 발사하여 R/Ø를 구분한다.

| 단계 | 처리 |
|:---:|------|
| 1 | K와 동일한 원 검출 |
| 2 | 원 중심에서 0°~315° 8방향 레이 발사 |
| 3 | 각 레이의 도면 경계 교차 거리 측정 |
| 4 | 거리 ≈ 반지름 → R(×2 변환), 거리 ≈ 직경 → Ø |

- **강점**: 원 검출과 부분적으로 독립 — 내원 감지에 유리
- **약점**: 도면 내 다른 형상이 레이를 차단하면 오판

## 디버그 오버레이 색상 범례

각 방법의 디버그 이미지에서 사용되는 색상:

### 공통 (K/L/N)

| 색상 | 요소 | 의미 |
|------|------|------|
| 🔵 파란색 원 | 외원 (Outer) | Contour 기반 외부 원 |
| 🩵 시안색 원 | 내원 (Inner) | 내부 보어 형상 |
| 🟢 초록색 선 | 치수선 | LSD 검출 치수 표기선 |
| 🔴 빨간색 라벨 | OD | 외경 분류 결과 |
| 🔵 파란색 라벨 | ID | 내경 분류 결과 |
| 🟢 초록색 라벨 | W | 폭 분류 결과 |

### K방법 전용

| 색상 | 요소 | 의미 |
|------|------|------|
| 🟠 주황색 사각형 | Focused ROI | 1.8×r 집중 OCR 영역 |
| 🟡 노란색 사각형 | Wide ROI | 3.0×r 확장 OCR 영역 |
| ⬜ 흰색 사각형 | OCR bbox | 텍스트 인식 위치 |

### L방법 전용 — 끝점 색상

| 색상 | 끝점 유형 | 의미 |
|------|-----------|------|
| 🔴 빨간색 점 | center | 원 중심 근처 끝점 → R(반지름) |
| 🟣 마젠타 점 | circumference | 원주 위 끝점 → Ø(직경) |
| 🟢 초록색 점 | unknown | 판정 불가 → 크기순 폴백 |
| 🟡 노란색 선 | 매칭 연결 | 치수 ↔ 치수선 매칭 |

### M방법 전용 — 심볼 색상

| 색상 | 기호 | 의미 |
|------|------|------|
| 🟣 마젠타 박스 | Ø | 직경 기호 검출 |
| 🟠 주황색 박스 | R | 반지름 기호 검출 (→ ×2 변환) |
| ⬜ 흰색 박스 | — | 기호 없는 일반 텍스트 |

### N방법 전용 — 레이캐스트 색상

| 색상 | 요소 | 의미 |
|------|------|------|
| 🌈 8색 레이 | 방향별 레이 | 0°~315° 각도별 고유 색상 |
| 🔵 파란색 원 | R 거리 | 반지름 거리 원 |
| 🟢 녹색 원 | D 거리 | 직경 거리 원 |
| 🟠 RADIUS zone | 교차 치수 | 0.7~1.3 × R 거리 |
| 🟣 DIAMETER zone | 교차 치수 | 1.7~2.3 × R 거리 |

## 4개 사례 × 4방법 정확도 매트릭스

| 방법 | Thrust (성공) | T5 (실패) | TD0062042 (랜덤) | Casing (단품) |
|------|:---:|:---:|:---:|:---:|
| **K** 기하학기반 | **3/3** | **3/3** ⬆️ | — | — |
| **L** 끝점토폴로지 | 2/3 | 0/3 | — | — |

### 사례별 상세 결과

| 도면 | GT (OD / ID / W) | K | L | M | N |
|------|-------------------|---|---|---|---|
| **Thrust** | 515 / 440 / 250 | 515 / 440 / 250 | 515 / 450 / 250 | Ø204 / 450 / 250 | 515 / 450 / 250 |
| **T5** | 1036 / 580 / 200 | **Ø1036 / 580 / 200** | 650 / `Ø70)`\* / 297.5 | 650 / `Ø70)`\* / 297.5 | 650 / 580 / 297.5 |
| **TD0062042** | (없음) | 635 / 200 / 57 | 200 / `Ø40)`\* / — | 200 / `Ø40)`\* / — | 200 / `Ø40)`\* / — |

## 핵심 인사이트

1. **원 검출이 기초**: K/L/N 모두 원 검출에 의존. 동심성 39px(Thrust)에서는 성공, 194px(T5)에서는 오류 전파.
2. **Ø 접두사 스케일 필터 완화**: T5에서 `Ø1036`이 스케일 필터(`r_outer × 1.2`)에 탈락했던 문제 → Ø 값은 `× 2.0`으로 완화하여 K방법 3/3 달성.
3. **폴백의 함정**: L/N은 끝점/레이 분석이 실패하면 크기순 폴백 → 결국 같은 결과를 냄.
4. **ASSY vs 단품**: Bearing Casing(단품)은 상세도 5개가 노이즈를 3배 증가시킴. K방법만 비교적 안정적.

## E09 방법론 다양화 (2026-03-23)

K방법 단독의 한계를 보완하기 위해 4개 신규 방법을 추가하고 앙상블을 구성했다.

### 신규 방법 (S01~S04)

| 방법 | 원리 | 원 검출 의존 |
|------|------|:---:|
| **S01** 화살촉 모폴로지 | Black Hat → 화살촉 쌍 → 치수선 복원 | 없음 |
| **S02** Dimension-Text-First | Ø 텍스트 우선 → LSD 역추적 → 방향 분류 | 없음 |
| **S03** Randomized Hough | 2점+법선 교차 → 어큐뮬레이터 투표 | 원 검출 전용 |
| **S04** Ellipse Decomposition | 컨투어 곡률 분해 → 호 그룹핑 → 원 복원 | 원 검출 전용 |

### 앙상블 v2 (K + S01 + S02 + S06) — 87개 도면 배치 결과

| 지표 | K 단독 | 앙상블 v1 | **앙상블 v2** |
|------|--------|----------|-------------|
| OD 검출 | 53/87 (61%) | 73/87 (84%) | **80/87 (92%)** |
| ID 검출 | 45/87 (52%) | 61/87 (70%) | **68/87 (78%)** |
| W 검출 | 42/87 (48%) | 59/87 (68%) | **69/87 (79%)** |
| **All 3** | **42/87 (48%)** | **52/87 (60%)** | **62/87 (71%)** |

- S01이 K의 W 미검출 보완에 효과적
- S02가 K의 OD 미검출 보완에 효과적
- S06(YOLO+OCR)이 27개 도면에서 추가 투표 기여
- 완전 실패 4개, 부분 검출 21개 — OD가 비정상인 경우 비-베어링 도면 추정

### 딥러닝 방법 (S05~S07) — 탐색 결과

| 방법 | 접근 | 결과 | 판정 |
|------|------|------|------|
| **S05** CircleNet-Lite | U-Net 0.5M + HoughCircles pseudo-label | 센터 피크 학습됨, 반지름 회귀 실패 | 보류 — dense label 필요 |
| **S06** YOLOv11-OBB | PaddleOCR bbox → YOLOv11n fine-tune (87개) | **mAP50 = 0.301**, P=0.40, R=0.28 | 학습 가능성 입증 |
| **S07** Florence-2 | base 0.23B zero-shot (OCR, Grounding, Caption) | 형상 이해 OK, OCR 3개만 검출 | 보류 — LoRA fine-tune 필요 |

S06이 OCR bbox 기반 pseudo-label로 학습 가능성을 입증. 더 많은 도면과 클래스 세분화로 개선 여지 있음.

### 앙상블 v3 (K-priority) — 검증 기반 개선

v2의 "71% All-3"은 값 존재만 카운트. GT 대비 검증 결과 투표가 K의 정확한 답을 덮어쓰는 문제 발견.

| 지표 | v2 (투표) | **v3 (K-우선)** |
|------|-----------|-----------------|
| All-3 검출 | 62/87 (71%) | 62/87 (71%) |
| **물리적 타당성** (OD>ID>W) | 36/87 (41%) | **49/87 (56%)** |
| **GT 정확도** (±5mm) | 3/6 (50%) | **5/6 (83%)** |

- **문제**: S01/S02가 노이즈 투입 → 투표에서 K의 올바른 ID 값을 덮어씀
- **해결**: K 결과가 물리적으로 타당(OD>ID>W)하면 K 우선 사용, K 실패 시에만 투표
- 전략 분포: K-priority 40개, vote-fallback 44개, K-partial 3개

## 사례별 상세 분석

각 도면에 K/L/M/N 4개 방법을 적용한 단계별 디버그 이미지와 분석:

- [Thrust Bearing — 성공 사례 (3/3)](./thrust/) — K방법이 완벽 동작하는 대표 케이스
- [T5 Bearing — 개선 사례 (3/3)](./t5/) — 스케일 필터 완화로 K방법 1/3 → 3/3 개선
- [TD0062042 — 라벨 없는 케이스](./random/) — GT 없이 4방법의 추출 결과 비교
- [Bearing Casing — 단품 도면 사례](./casing/) — ASSY가 아닌 단품 가공도에서의 기하학 방법 동작 비교
