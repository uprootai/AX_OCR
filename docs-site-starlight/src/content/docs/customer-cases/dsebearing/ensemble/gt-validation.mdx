> 2개 ASSY GT 도면의 상세 검증 결과와 GT 확보의 한계를 분석한다.

## 확보된 GT (2개)

87개 도면 중 ASSY 도면 설명에 정확한 치수가 명시된 **2개만** GT로 사용 가능하다.

:::caution[통계적 한계]
n=2는 통계적으로 유의미한 표본이 아니다. "GT 정확도 100%"는 **2개 ASSY 도면에 한정된 결과**이며, 전체 87개 도면의 정확도를 대표하지 않는다. 단품(~50개)과 비베어링(~25개) 도면은 GT 없이 물리적 타당성(OD>ID>W)과 soft filter confidence로만 품질을 추정한다.
:::

| GT-2 | TD0062055 | RADIAL BEARING | Ø515 | 440 | 48 |

## GT-1: TD0062037 — Thrust Bearing

Dimension Lab에서 TD0062037을 선택하면, GT가 설정된 상태로 표시된다: **OD(외경)=1036, ID(내경)=580, W(폭)=200**.

![GT-1 도면 선택 — GT 설정 완료 (OD=1036, ID=580, W=200)](/img/ensemble/gt1-with-gt-set.png)

### 버전별 결과

| 버전 | OD | ID | W | 정확 |
|------|:---:|:---:|:---:|:---:|
| v2 (투표) | Ø1036 ✅ | 580 ✅ | 200 ✅ | 3/3 |
| v3 (K-priority) | Ø1036 ✅ | 580 ✅ | 200 ✅ | 3/3 |
| v5 (최적화) | Ø1036 ✅ | 580 ✅ | 200 ✅ | 3/3 |

이 도면은 **모든 버전에서 정확**했다. K 방법이 원형 구조를 잘 인식하는 전형적인 ASSY 도면.

### W 선택 과정 (v5)

- `expected_w = (1036 - 580) / 2 = 228`
- W 후보 중 200은 `|200 - 228| / 228 = 0.12` → 50% 범위 내 → radial 공식으로 선택
- v3에서도 200이 선택됐으므로 변경 없음

### 방법별 결과

| 방법 | OD | ID | W |
|------|:---:|:---:|:---:|
| K | Ø1036 ✅ | 580 ✅ | 200 ✅ |
| S01 | Ø1036 ✅ | 650 ❌ | 200 ✅ |
| S02 | Ø1036 ✅ | 650 ❌ | 200 ✅ |

S01/S02는 ID를 650으로 잘못 분류. 원 proximity 없이는 580 vs 650 구분이 어렵다.

7×14 매트릭스에서 K 기하학 방법이 **전 엔진 100%** 정확도를 달성한다. Best Top 5도 모두 K.기하학기반 × 100%.

![GT-1 매트릭스 — K.기하학기반 전 엔진 100%, Best Top 5 모두 Ø1036/580/200](/img/ensemble/gt1-matrix-with-gt.png)

## GT-2: TD0062055 — Radial Bearing

Dimension Lab에서 TD0062055를 선택하면, GT가 설정된 상태로 표시된다: **OD(외경)=515, ID(내경)=440, W(폭)=48**.

![GT-2 도면 선택 — GT 설정 완료 (OD=515, ID=440, W=48)](/img/ensemble/gt2-with-gt-set.png)

### 7×14 매트릭스 결과

K.기하학기반이 **전 엔진 100%** 정확도를 달성한다. OD/ID는 다수 방법에서 100% 일치하나, W=48을 정확히 검출하는 것은 K 방법 뿐이다.

![GT-2 매트릭스 — K.기하학기반 전 엔진 100%, OD/ID 다수 방법 100%](/img/ensemble/gt2-matrix-with-gt.png)

### 버전별 결과

| 버전 | OD | ID | W | 정확 |
|------|:---:|:---:|:---:|:---:|
| v2 (투표) | Ø515 ✅ | 440 ✅ | **250** ❌ | 2/3 |
| v3 (K-priority) | Ø515 ✅ | 440 ✅ | **250** ❌ | 2/3 |
| v5 (최적화) | Ø515 ✅ | 440 ✅ | **48** ✅ | **3/3** |

**v5에서 W가 수정된 핵심 사례.**

### W 선택 과정 비교

**v3 (잘못된 결과)**:
- `ID/OD = 440/515 = 0.854 > 0.75` → 스러스트 경로
- 스러스트: 최대 W 선택 → **W=250**

**v5 (올바른 결과)**:
- `expected_w = (515 - 440) / 2 = 37.5`
- W 후보: 250, 90, 48.5, ...
- 48.5: `|48.5 - 37.5| / 37.5 = 0.29` → 50% 범위 내 ✅
- 250: `|250 - 37.5| / 37.5 = 5.67` → 범위 밖 ❌
- radial 공식으로 **W=48** 선택 (48.5 반올림)

### 방법별 결과

| 방법 | OD | ID | W |
|------|:---:|:---:|:---:|
| K | Ø515 ✅ | 440 ✅ | 48 ✅ (v5) |
| S01 | Ø515 ✅ | 440 ✅ | — |
| S02 | Ø515 ✅ | 440 ✅ | — |

S01/S02는 W를 검출하지 못했다. K만이 W를 제공하므로 K-priority의 W 분류 정확도가 핵심.

## GT 확보의 한계

### BOM size 필드 ≠ GT

초기에 BOM `bom_items.json`의 `size` 필드에서 22개 GT 후보를 추출했다.

예시:
| 도면 | BOM size | 앙상블 OD | 일치 |
|------|---------|-----------|:---:|
| TD0062039 | 660×350×130 | Ø635 | ❌ |
| TD0062046 | 290×210×30 | Ø260 | ❌ |

분석 결과, BOM size는 **소재 규격**(가공 전 원자재 치수)으로 확인됐다.
완제품 치수와 수 mm~수십 mm 차이가 나므로 직접 GT로 사용할 수 없다.

### GT 확보 방법

| 방법 | 난이도 | 신뢰도 | 현실성 |
|------|:---:|:---:|:---:|
| ASSY 도면 설명 텍스트 | 낮음 | 높음 | ~7개 (현재 2개 확보) |
| 고객 제공 치수표 | 낮음 | 최고 | 고객 협조 필요 |
| 수동 측정 (도면 눈대중) | 중간 | 중간 | 87개 전수 가능하나 노동집약 |
| BOM size 범위 검증 | 낮음 | 낮음 | ±10% 범위 체크용으로만 |

## 향후 과제

### 단기 (GT 확보)

1. **나머지 5개 ASSY 도면** GT 추출 시도
2. 고객에게 주요 단품 치수표 요청
3. BOM size 기반 ±10% 범위 검증 자동화

### 중기 (정확도 개선)

1. **입력 다양화**: K와 독립적인 OCR 파이프라인 (현재 fallback OCR로 부분 구현)
2. **S06 YOLO 강화**: 더 많은 학습 데이터 + 클래스 세분화
3. **Florence-2 LoRA**: GD&T 커스텀 태스크 fine-tune

### 장기 (시스템 개선)

1. **Dimension Lab UI에 GT 입력 기능** 추가
2. 고객이 직접 GT를 입력하고 정확도를 실시간 추적하는 피드백 루프
3. GT 축적에 따른 자동 재평가 + 방법론 가중치 학습
