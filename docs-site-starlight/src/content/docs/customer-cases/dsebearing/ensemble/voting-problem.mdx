> v2 순수 투표에서 S01/S02 노이즈가 K의 정답을 덮어쓴 구조적 문제와 해결 과정을 분석한다.

## 문제 발견

v2 앙상블은 K, S01, S02, S06 네 방법의 결과를 **다수결 투표**로 결합했다.
87개 도면에서 All-3 검출률 71%를 달성했지만, GT 대비 정확도를 측정하자 **50%**에 불과했다.

### TD0062055 사례 — K가 맞았는데 투표가 틀림

![GT-2 (TD0062055) 도면 — Radial Bearing ASSY](/img/ensemble/gt2-with-gt-set.png)

| 방법 | OD | ID | W |
|------|----|----|---|
| **K (geometry)** | Ø515 ✅ | 440 ✅ | 250 ❌ |
| S01 (arrowhead) | Ø515 ✅ | 440 ✅ | 250 ❌ |
| S02 (text-first) | Ø515 ✅ | 440 ✅ | — |
| **투표 결과** | Ø515 ✅ | 440 ✅ | **250** ❌ |
| **GT** | Ø515 | 440 | **48** |

W=250은 세 방법 모두 동의했으므로 투표는 250을 선택했다.
하지만 **실제 W=48**. S01/S02가 K와 같은 오류를 공유하면 투표로 수정이 불가능하다.

## GT-1 (TD0062037) 매트릭스 — K 방법의 우위

TD0062037(Thrust Bearing)의 7×14 매트릭스를 보면, K.기하학기반만 **전 엔진 100%** 정확도를 달성한다. 다른 방법은 0~67%에 그친다. 이것이 K-priority 전략의 근거다.

![GT-1 매트릭스 — K.기하학기반 전 엔진 100%, 나머지 방법 0~67%](/img/ensemble/gt1-matrix-with-gt.png)

## 근본 원인: 방법 간 종속성

S01, S02는 **K의 중간 결과**(dims, circles)를 입력으로 받는다.

```
K 실행 → dims 캡처 → S01(dims) → S02(dims)
```

K가 추출한 치수 목록을 S01/S02가 재분류하는 구조이므로, **K의 OCR이 잘못되면 S01/S02도 같은 입력에서 같은 오류를 생산**한다.

이 경우 "독립적인 3개 방법의 다수결"이 아니라 "같은 입력에 대한 3가지 해석"에 불과하다.

### 구조적 버그: K 실패 시 S01/S02 무력화

더 심각한 문제도 발견됐다. K의 원 검출이 실패하면:

1. K 내부의 `_classify_by_circle_proximity` 함수가 호출되지 않음
2. monkey-patch 훅이 실행되지 않아 `captured_dims`가 빈 리스트
3. S01/S02에 전달할 치수가 없어 **아예 실행되지 않음**

87개 도면 중 **31개**가 이 경우에 해당했다. 이 도면들은 S06(YOLO)만 결과를 제공하거나, 아무 결과도 없었다.

## 해결: K-priority 전략

### 전략 설계

K 방법이 87개 도면 중 가장 정확하다는 실증 데이터를 기반으로, **K 우선** 전략을 도입했다.

```
if K가 OD > ID > W를 모두 만족 (물리적 타당):
    → K 결과 그대로 사용 (k_priority)
elif K가 OD + ID만 유효:
    → K의 OD/ID 사용, W만 투표 (k_partial)
else:
    → 기존 투표 (vote_fallback)
```

### 물리적 타당성 검증

베어링의 물리적 특성상 반드시 **OD > ID > W**여야 한다.

| 검증 | 조건 | 위반 시 |
|------|------|---------|
| 크기 순서 | OD > ID > W | K 결과 신뢰 불가 → 투표 |
| 범위 | 50 < OD < 2000mm | 비베어링 가능성 |
| 비율 | ID/OD < 0.95 | 내외경 구분 의미 있음 |

### K 방법의 기하학 기반 분류

K 방법은 원 검출 + proximity 기반으로 OD/ID/W를 분류한다. 14개 방법 중 **K.기하학기반만 100%**, 나머지는 50% 이하임을 방법별 정확도 패널에서 확인할 수 있다.

![GT-1 방법별 정확도 — K.기하학기반 100%, A.Ø기호+크기 50%](/img/ensemble/gt1-methods-accuracy.png)

### 적용 결과

| 전략 | v2 | v3 |
|------|----|----|
| k_priority | 0 | **40** |
| k_partial | 0 | 3 |
| vote_fallback | 87 | 44 |
| GT 정확도 | 50% | **83%** |

K-priority만으로 GT 정확도가 50%→83%로 상승했다.

## 교훈

1. **종속적 방법의 투표는 독립 투표가 아니다** — 같은 OCR 결과를 공유하면 오류도 공유
2. **방법 수보다 입력 다양성이 중요** — 4개 방법이 같은 입력이면 1개와 동일
3. **가장 정확한 방법 우선, 투표는 보완용** — 앙상블의 목적은 K의 부족분을 채우는 것
