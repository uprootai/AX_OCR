### 핵심 발견: BOM 색깔 구분

1차 미팅 최대 성과 — BOM의 색깔이 **견적 대상 여부**를 결정한다는 것을 확인했습니다.

| BOM 색깔 | 의미 | 수량 | 견적 처리 |
|---------|------|------|----------|
| 🟥 분홍색 | 최종 조립도 (Assembly) | 7개 | **스킵** — 치수 추출 불필요 |
| 🟦 파란색 | 중간 조립품 (Sub-assembly) | 16개 | **스킵** — 치수 추출 불필요 |
| ⬜ **하얀색** | **단품 도면 (Part)** | **303개** | **견적 대상** — 각각 견적 산출 후 합산 |

:::tip[김동엽 대표 원문]
*"하얀색으로 표기돼 있는 도면들을 개개별로 소재 치수부터 시작해서 단품의 견적이 다 나와야지 합산 견적이 나오거든요."*
:::

### 도면 영역별 추출 필요성 (김동엽 대표 확인)

| 도면 영역 | 필요 여부 | 김 대표 설명 |
|----------|----------|-------------|
| **큰 소재 사이즈** (Ø550 등) | **1순위 필수** | *"이 소재의 치수가 제일 1번으로 빠져야 됨"* |
| 구멍 수, 가공 형태 | 2순위 (나중) | *"잔잔한 구멍들이 작업에 따라 가공비가 추가로 들어감"* |
| 각도 정보 | 2순위 (나중) | *"소재 견적 빠지고 나면 2차로 가공 견적 뺄 때 참고"* |
| 가공 표기 (표면처리 등) | 참고 | *"요 부위에 대한 가공을 어떻게 해라는 표기들"* |
| 상단 공차 테이블 | **불필요** | *"일반적인 공차 표시. 견적상 크게 의미 없음"* |
| NOTES / ISO VIEW | **불필요** | 확인 결과 불필요 |
| SECTION 상세 | 2순위 (나중) | *"한참 뒤에 접근해야 되는 부분"* |

### 작업 범위 확정

| 자료 | 사용 여부 | 이유 |
|------|----------|------|
| BOM | **사용** | 소재 재질, 도면번호, 부품 목록 기준 |
| 단품 도면 (하얀색) | **사용** | 치수, 재질, 가공 정보 추출 대상 |
| 조립도 (분홍/파란색) | **불필요** | *"수치보다는 조립 내용을 표현하는 도면"* |
| 사양서 (STMPS00095) | **불필요** | *"BOM에 재질 다 있으니까 그걸로 충분"* |
| 품질기준서 (TD0060732) | **불필요** | *"견적 볼 때는 안 봐도 되는 부분"* |
