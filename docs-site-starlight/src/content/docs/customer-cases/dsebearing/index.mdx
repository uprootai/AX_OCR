> 터빈 베어링 1세트 견적(12일 소요) 중 사양 확인 4일을 자동화로 단축하는 도면 분석·견적 자동화 사례입니다.

## 고객 정보

| 항목 | 내용 |
|------|------|
| 회사명 | (주)동서기연 |
| 업종 | 터빈/발전기 베어링 제조 |
| 주요 거래처 | 두산에너빌리티 |
| 기간 | 2026-01 ~ 진행 중 |
| 도면 유형 | 기계 부품도 (ISO/JIS 규격) |

<Aside type="note" title="용어 안내">
- **BOM** (Bill of Materials) — 부품 명세서. 제품을 구성하는 모든 부품의 목록입니다.
- **ASSY / SUB / PART** — BOM 계층. ASSY(최종 조립품) → SUB(중간 조립품) → PART(개별 단품, 견적 대상)
- **치수** — 도면에 기재된 크기 정보 (외경, 내경, 길이 등)
- **eDOCr2** — 도면에서 텍스트와 치수를 자동 추출하는 OCR 엔진입니다.
</Aside>

## 문제: 1세트 견적에 12일

터빈 베어링 1세트를 만들려면 수백 개의 부품이 필요합니다. 기존에는 도면을 하나하나 열어 크기를 확인하고, 재질에 맞는 원재료 가격을 조회한 뒤, 가공비를 더해 견적을 산출했습니다.

| 단계 | 소요일 | 내용 |
|------|--------|------|
| 사양 확인 | 4일 | 도면마다 재질과 크기를 눈으로 확인 |
| 외주 견적 | 6일 | 협력사에 견적 요청 후 취합 |
| 자체 검토 | 2일 | 산출 결과를 사람이 최종 확인 |
| **합계** | **12일** | |

<Aside type="tip" title="고객 원문">
*"사양 확인하고 도면에 큰 치수에 여유치를 붙여서 일단 소재 견적을 받아야 되는 부분이거든요."*

**풀이**: 원재료(소재)를 주문할 때는 도면의 최종 크기보다 약간 크게 주문합니다. 이 추가분을 **여유치**라고 합니다. 가공 중 깎아내는 양을 미리 감안하는 것입니다.
</Aside>

## 해결 방향: 자동화 파이프라인

12일 중 **사양 확인 4일**을 자동화로 단축하는 것이 1차 목표입니다.

<Aside type="caution" title="2차 미팅 방향 전환 (03-17)">
1차 미팅에서는 "큰 소재 사이즈"라고만 했기 때문에 모든 치수를 추출하는 방향으로 진행했으나, 2차 미팅에서 **외경(OD)·내경(ID)·폭(Width) 3개만** 리스트화하면 된다고 확정되었습니다. 기존 OCR 전체 추출 시스템은 유지하되, 앞단에 **OD/ID/폭 분류기**를 추가합니다. → [2차 미팅 상세](./2nd-meeting)
</Aside>

### 단계별 설명

| 단계 | 내용 | 자동화 | 비고 |
|------|------|--------|------|
| **1. 자료 수령** | 94개 PDF (BOM 1 + 도면 87 + 사양서·품질기준서 6) | - | |
| **2. BOM 계층 분류** | Excel BOM의 행 배경색으로 계층 분류: 🩷ASSY(9) → 🟨SUB(19) → ⬜PART(361) | ✅ 자동 | |
| **3. 단품 도면 매칭** | ⬜PART 361개 중 도면이 확보된 품목을 BOM과 매칭 | ✅ 자동 | |
| **4. OCR 전체 추출** | 매칭된 단품 도면에서 모든 치수를 OCR로 추출 | ✅ 자동 | 기존 시스템 유지 |
| **5. OD/ID/폭 분류** | 추출된 치수 중 외경·내경·폭 3개를 자동 판별 | ✅ v7 완료 | GT 7건 **90%** (19/21) |
| **6. 소재비 산출** | OD/ID/폭 → 재질별 소재 사이즈 결정 → 소재비 산출 | ✅ 자동 | 1단계 견적 |
| **7. 홀 개수 파악** | 도면에서 Ø 마크 기반 홀 개수 자동 파악 | ⬜ 2단계 | 가공비 산출용 |

<Aside type="caution" title="왜 BOM 분류가 먼저인가?">
조립품(🩷ASSY, 🟨SUB)은 하위 단품의 합산이므로, 조립도에서 치수를 추출하면 **이중 계산**됩니다. BOM에서 먼저 ⬜PART(단품)를 선별한 뒤, 해당 단품의 도면만 OCR 대상으로 넘겨야 합니다.
</Aside>

### 핵심 수치

| 단계 | 지표 | 값 | 비고 |
|------|------|-----|------|
| 자료 수령 | 수령 PDF | **94개** | 터빈 베어링 1세트 전달 자료 패키지 |
| BOM 분류 | 등록 품목 | **389개** | 🩷ASSY 9 + 🟨SUB 19 + ⬜PART 361 |
| 도면 매칭 | 견적 대상 | **⬜361개** | 이 중 현재 도면 확보: 11종 |
| OCR 추출 | 전체 치수 | **548개** | 11종 도면에서 전체 추출 (기존 시스템) |
| **OD/ID/폭** | 핵심 치수 | **부품당 3개** | 2차 미팅 확정 — 외경·내경·폭 자동 분류 (**GT 7건 기준 90% 정확도**) |
| 견적 산출 | 산출 금액 | **₩3,200,466** | 11종 54개 품목 (361개 중 15.0%), 내부 추정 단가 |

<Aside type="note" title="진행 흐름 요약">
94개 자료 패키지를 수령하여 BOM Excel에서 389개 품목을 계층 분류했습니다. 견적 대상은 ⬜PART 361개입니다. 2차 미팅에서 **외경·내경·폭 3개만** 필요하다고 확정되어 앙상블 분류기를 개발했습니다. GT 7건 ASSY 도면 기준 **OD 7/7, ID 7/7, W 5/7 = 90% 정확도**를 달성했습니다. 80개 Non-GT 도면 회귀 테스트에서도 에러 0건, 물리적 순서 위반 2건(소형 부품)으로 안정적입니다.
</Aside>
