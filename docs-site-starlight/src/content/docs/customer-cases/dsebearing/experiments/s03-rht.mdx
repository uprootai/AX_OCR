> HoughCircles의 파라미터 민감성과 부분 호 한계를 극복하는 투표 기반 원 검출.

## 핵심 아이디어

기존 원 검출(Contour, HoughCircles, RANSAC)은 에지 기반이라 노이즈에 민감하다.
RHT는 2점 + gradient 방향으로 원 중심을 투표하여 **부분 호에서도 원을 검출**한다.

### RANSAC과의 차이

| 항목 | RANSAC | RHT |
|------|--------|-----|
| 접근 | 3점 샘플 → 모델 → inlier 검증 | 2점 + gradient → 투표 |
| 복잡도 | O(N) per iteration | O(1) per iteration |
| 부분 호 | 간접 (inlier ratio) | 직접 (arc coverage) |
| 다중 원 | 순차 제거 필요 | 어큐뮬레이터 피크 추출 |

## 구현

```
에지 포인트에서 2점 랜덤 샘플링
  → Sobel gradient 방향으로 원 중심 후보 계산
  → 어큐뮬레이터 (cx, cy) 공간에 투표
  → 임계값(vote ≥ 3) 이상 피크 추출
  → 호 커버리지 > 30%이면 유효
```

설정: `max_iter=15000`, `vote_threshold=3`

## 결과

### 원 검출 수

| 도면 | RANSAC | **RHT** | Decomp (S04) |
|------|:------:|:-------:|:-----------:|
| TD0062037 (T5) | 553 | **18** | 46 |
| TD0062042 (T6) | 478 | **20** | 41 |
| TD0062019 (T7) | 2415 | **8** | 33 |

**RHT가 가장 정제된 결과를 반환한다.** RANSAC은 수백~수천 개의 노이즈 원을 생성하지만, RHT는 투표 임계값으로 8~20개로 자연스럽게 필터링된다.

### OD/ID/W 직접 기여

RHT 단독으로는 OD/ID/W를 분류하지 않는다. 원 검출 결과를 K방법 파이프라인에 공급하는 **보조 원 검출기** 역할.

현재 앙상블에서는 S03 원 검출이 독립적으로 투표에 참여하지 않고, K방법 내부의 원 검출 후보로 활용된다.

## 한계

| 한계 | 설명 |
|------|------|
| gradient 노이즈 | 스캔 품질이 낮으면 Sobel gradient가 부정확 |
| bin size 튜닝 | 어큐뮬레이터 해상도에 따라 결과 변동 |
| 직접 분류 안 됨 | 원만 검출, OD/ID/W 분류는 별도 필요 |
