> 화살촉을 직접 찾아 "이 텍스트가 어디를 가리키는지" 역추적한다. W 검출 보완에 가장 효과적.

## 핵심 아이디어

K방법은 원 검출 → 스케일 추정 → 크기순 분류로 **간접 추론**한다.
화살촉 검출은 반대로 **치수선 → 텍스트 → 측정 대상**을 직접 매핑한다.

```
화살촉 2개 → 치수선 → 양 끝 위치 = 측정 대상
치수선 근처 텍스트 = 해당 치수값
치수선이 원 직경 방향 → OD/ID 후보
치수선이 수직 방향 → W 후보
```

## 파이프라인

| 단계 | 입력 | 처리 | 출력 |
|------|------|------|------|
| 1. 화살촉 검출 | 이진화 도면 | Black Hat (closing - original) | 화살촉 위치 + 방향 |
| 2. 쌍 매칭 | 화살촉 리스트 | 동일 직선 위 쌍 찾기 (±10°) | 치수선 리스트 |
| 3. 텍스트 매칭 | 치수선 + OCR | 중점에서 가장 가까운 텍스트 | (치수선, 값) 쌍 |
| 4. OD/ID/W 분류 | 매칭 결과 + 원 | 방향 분석 → 직경 vs 축 | OD, ID, W 값 |

### 화살촉 검출 조건

- 면적: 50~500px²
- 종횡비: 0.3~3.0
- Black Hat으로 실선 삼각형 화살촉 검출
- Ø 접두사가 있는 텍스트는 원 검출 없이도 OD 후보로 직접 분류

## 결과

### 87개 배치 기여

| 지표 | K 단독 | K + S01 |
|------|:------:|:-------:|
| W 검출 | 42/87 (48%) | **59/87 (68%)** |
| 개선 | — | **+17** |

**S01은 K가 W를 못 찾은 도면에서 가장 효과적이다.** 치수선의 수직 방향 분석이 W 후보를 직접 제공한다.

### T5 도면 (GT: OD=1036, ID=580, W=200)

| 값 | S01 결과 | 판정 |
|----|---------|:---:|
| OD | Ø1036 | ✅ |
| ID | 650 | ❌ (580이 정답) |
| W | 200 | ✅ |

ID=650 오류: 원 proximity 없이는 580 vs 650을 구분할 수 없다. 화살촉만으로는 "어느 원의 직경인지"를 판단하기 어렵다.

## 한계

| 한계 | 설명 |
|------|------|
| ID 구분 불가 | 580 vs 650 같이 크기가 비슷한 직경은 원 proximity 없이 구분 안 됨 |
| 열린 화살촉 | `>---<` 형태는 Black Hat으로 검출 어려움 |
| 참조선 혼동 | leader line의 화살촉과 치수선 화살촉이 섞임 |

## 앙상블 역할

앙상블 v2에서 **투표 참여자**로 등록. K가 W를 반환하지 않을 때 S01의 W가 fallback으로 선택된다. 87개 중 17개 도면에서 W를 보완.
