> 7가지 방법론의 실험 결과를 비교하고, 앙상블에 어떻게 기여했는지 정리한다.

### End-to-End 파이프라인

![E2E 파이프라인](/images/experiments/e2e_pipeline.jpg)

PDF 업로드 → 7개 OCR 엔진 → 원 검출(K+RHT+Ellipse) → 분류(투표/SEC) → OD/ID/W → 견적. ASSY 도면은 SEC-priority(v6), 단품은 K-priority(v5).

### 왜 K만으로는 부족한가

![동기 체인 — K 한계가 만든 실험들](/images/experiments/motivation_chain.jpg)

K의 4가지 한계가 각각 S01~S04와 SEC를 만들었다. 최종 결과: K 단독 0/7(ASSY) → **앙상블 v7 19/21(90%)**.

## 배경

기존 K방법(기하학 기반)은 원 검출 → 스케일 추정 → 크기순 분류로 OD/ID/W를 추론한다.
87개 도면 배치에서 K 단독 정확도는 **48% (All-3)**. 원 검출 실패 시 전체 파이프라인이 무너지는 구조적 한계가 있다.

이를 보완하기 위해 7가지 대안 방법론을 실험했다.

## 방법론 분류

| 분류 | 방법 | 핵심 아이디어 | GPU |
|------|------|-------------|:---:|
| **규칙 기반** | S01 화살촉 | 모폴로지로 화살촉 검출 → 치수선 복원 | 없음 |
| | S02 Text-First | OCR에서 Ø 먼저 → 치수선 역추적 | 없음 |
| **원 검출 보강** | S03 RHT | Randomized Hough Transform | 없음 |
| | S04 Ellipse | 컨투어 호+직선 분해 → 그룹핑 | 없음 |
| **딥러닝** | S05 CircleNet | U-Net 히트맵 → 원 센터 예측 | ~0.1GB |
| | S06 YOLO-OBB | OCR bbox pseudo-label → 치수 영역 검출 | ~4GB |
| | S07 Florence-2 | VLM zero-shot → 형상 이해 | ~1GB |

| **K (baseline)** | 53/87 | 42/87 | 주력 (k_priority) | ✅ |
| **S01 화살촉** | — | **+17** | W 미검출 보완에 효과적 | ✅ |
| **S02 Text-First** | **+20** | — | OD 미검출 보완에 효과적 | ✅ |
| **S06 YOLO** | **+7** | **+10** | 27/87 도면에서 투표 참여 | ✅ |
| S03 RHT | — | — | 원 검출 참고 (8~20개) | ✅ |
| S04 Ellipse | — | — | 원 검출 참고 (33~46개) | ✅ |

### 앙상블 진화

| 버전 | 구성 | GT 기준 | GT 정확도 |
|------|------|:-----:|:---------:|
| K 단독 | K만 | 2건 | — |
| v2 | K + S01 + S02 + S06 투표 | 2건 | 3/6 (50%) |
| v5 | v2 + K-priority + fallback OCR + 필터 | 2건 | 6/6 (100%) |
| v6 | v5 + SEC-priority (SECTION 세로스캔) | 7건 | 15/21 (71%) |
| **v7** | v6 + 볼트필터 + 적응형이진화 + plausibility | **7건** | **19/21 (90%)** |

v5까지는 GT 2건(6개 비교점)으로 100%였으나, GT를 7건(21개 비교점)으로 확대하자 실제 정확도가 드러났다. v7에서 SEC 노이즈 필터 + OD 교차검증 + ID 적응형 재시도를 적용하여 **90%**를 달성했다.

| 항목 | v6 (GT 7건) | **v7 (GT 7건)** |
|------|:-----------:|:---------------:|
| OD | 5/7 | **7/7** |
| ID | 0/7 | **7/7** |
| W | 5/7 | 5/7 |
| 합계 | 10/21 (48%) | **19/21 (90%)** |
