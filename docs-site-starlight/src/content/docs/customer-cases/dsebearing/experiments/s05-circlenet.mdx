> 합성 원 데이터 1000개 + dense circle mask로 학습. 이전 pseudo-label의 반지름 회귀 실패를 해결.

## v1 — pseudo-label (실패)

| 항목 | 값 |
|------|---|
| 모델 | U-Net (0.5M params, 1.8MB) |
| 학습 데이터 | 30개 도면 (25 train / 5 val) |
| 라벨 | HoughCircles 기반 **pseudo-label** (sparse) |
| 학습 시간 | 34초 (50 epochs) |

| 지표 | 값 | 판정 |
|------|---|:---:|
| val loss (focal) | 0.0165 | 수렴 |
| 원 센터 검출 | 피크 5~25개 | GT 근처 |
| **반지름 회귀** | **0** (전부 0 예측) | **실패** |

**실패 원인**: HoughCircles pseudo-label은 원 중심에만 점을 찍는다. 반지름 정보가 sparse하여 U-Net이 학습 불가.

---

## v2 — 합성 dense mask (성공)

### 핵심 아이디어

87개 도면에서 원 영역을 크롭 → 랜덤 스케일/회전/배치 → 원 내부를 채운 **dense mask** label 생성.
pseudo-label이 아닌 **GT가 확실한 합성 데이터**.

### 합성 데이터 생성

![합성 학습 데이터 샘플 (빨간 = 원 마스크)](/images/experiments/s05_synthetic_samples.jpg)

| 항목 | 값 |
|------|---|
| 원 크롭 소스 | 87개 도면에서 HoughCircles → **370개** 크롭 |
| 합성 이미지 | **1000개** (이미지 + dense mask) |
| 이미지당 원 수 | 1~3개 (랜덤 배치) |
| 배경 | 도면 느낌 노이즈 (랜덤 선) |
| 반지름 범위 | 20~80px (랜덤) |
| 증강 | 90°/180°/270° 회전 |

### 학습 설정

| 항목 | 값 |
|------|---|
| 모델 | **TinyUNet (483K params, 1.8MB)** |
| Loss | BCE + Dice |
| 학습 데이터 | 900 train / 100 val |
| Epochs | 80 |
| Batch | 16 |
| Optimizer | Adam + CosineAnnealing |
| VRAM | **~0.1GB** |
| 학습 시간 | **325초** |

### 결과

| 지표 | v1 (pseudo) | v2 (256px) | **v2 (512px)** |
|------|:---:|:---:|:---:|
| params | 500K | 483K | **1.9M** |
| val loss | 0.0165 | 0.0022 | **0.0017** |
| 반지름 회귀 | 0 (실패) | 작동 | **작동** |
| 합성 데이터 | — | 1000개 | **1000개 (동심원+텍스트 노이즈)** |
| 학습 시간 | 34s | 325s | **1742s** |

**핵심 성과**: 반지름 회귀 실패 → 성공. Dense mask label + 동심원 합성 데이터가 문제를 해결.

### v3 — 실제 도면 배경 + negative sample

v2의 문제: 합성 데이터(흰 배경)에서는 val=0.0017로 수렴하지만, 실제 도면에서 87.8%를 원으로 예측. **도메인 갭**.

v3 변경:
- 배경을 실제 도면 랜덤 크롭으로 교체 (30개 도면 × 5 = 150개)
- 50%는 원이 없는 negative sample 포함

| 지표 | v2 (흰 배경) | **v3 (실제 배경)** |
|------|:---:|:---:|
| 실제 도면 >0.5 비율 | 87.8% | **45.2%** |
| 검출 원 수 | 1개 | **3개** |

개선되었으나 여전히 45%가 원으로 예측 — 실용 수준 아님.

### 분석 — 근본 한계

1. **해상도**: 원본 3306×2339px → 512px. 베어링 원(반지름 ~200px)이 ~30px로 축소
2. **SECTION에 원이 없음**: 단면도는 직사각형 프로파일. 원은 정면도에만 존재
3. **정면도 원 검출은 K가 이미 수행**: HoughCircles + Contour 기반으로 동작 중
4. **Codex GPT-5.4 vision이 T1/T5 모두 3/3**: 단일 API 호출로 원 검출 없이 직접 치수 추출

### 결론

| 항목 | 판정 |
|------|------|
| 반지름 회귀 실패 해결 | **성공** — dense mask label이 핵심 |
| 합성 데이터 학습 가능성 | **입증** — val loss 수렴 |
| 실제 도면 원 검출 | **실패** — 도메인 갭 + 해상도 한계 |
| 실용적 가치 | **낮음** — K방법 + Codex vision이 더 효과적 |

S05는 **"합성 데이터로 DL 원 검출기를 학습할 수 있다"는 가능성을 입증**했지만, 현재 파이프라인에서 K방법(기존)과 Codex vision(신규)이 더 실용적이다.
