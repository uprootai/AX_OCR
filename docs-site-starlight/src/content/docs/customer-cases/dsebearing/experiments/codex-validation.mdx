> 앙상블 v5의 코드와 결과를 Codex(GPT-5.4)에 넘겨 독립적으로 검증받은 결과.

![Codex 교차검증 결과](/images/experiments/codex_validation.jpg)

## 검증 방법

앙상블 v5의 핵심 코드(`dimension_ensemble.py`, `geometry_guided_extractor.py`)와 87개 배치 결과를 Codex에 전달하여:
1. 구조적 버그 탐색
2. GT 확장 가능성 검토
3. 비-베어링 필터링 전략 리뷰

## 발견 사항 4가지

### 1. vote-fallback 구조적 버그

![K 성공 vs 실패 도면 비교](/images/experiments/codex_k_success_fail.jpg)

좌: K 성공 (T1 ASSY — 녹색 원 검출) / 우: K 실패 (HEX BOLT — 원이 잡혀도 무의미). K가 실패하면 S01/S02도 입력이 없어 멈춘다.

![vote-fallback 버그 흐름도](/images/experiments/codex_bug_flowchart.jpg)

K 원 검출이 실패하면 monkey-patch 훅이 실행되지 않아 **S01/S02가 아예 작동하지 않는** 31개 도면이 있었다.

| 상태 | 도면 수 | S01/S02 |
|------|:------:|:-------:|
| K 성공 | 56 | 정상 동작 |
| **K 실패** | **31** | **미작동** |

v5에서 fallback OCR(`DimensionService`)을 추가하여 K 실패 도면에도 치수를 공급하는 것으로 **우회**했지만, 근본 수정(monkey-patch 의존 제거)은 미완.

:::caution[근본 원인 미수정]
현재 fallback OCR은 우회책이다. K 실패 시 S01/S02의 파이프라인이 정상 작동하도록 monkey-patch 의존을 제거하는 리팩토링이 필요하다.
:::

### 2. BOM size 필드 — GT가 아니다

![BOM size vs 실제 치수 비교](/images/experiments/codex_bom_comparison.jpg)

BOM `bom_items.json`의 `size` 필드에서 22개 GT 후보를 발견했으나 분석 결과:

| 도면 | BOM size | 앙상블 OD | 일치 |
|------|---------|-----------|:---:|
| TD0062039 | 660×350×130 | Ø635 | ❌ |
| TD0062046 | 290×210×30 | Ø260 | ❌ |

**BOM size는 소재 규격(가공 전 원자재 치수)**으로 확인. 완제품 치수와 수 mm~수십 mm 차이가 나므로 직접 GT로 사용할 수 없다.

**활용 방법**: ±10% 범위 검증으로만 사용. 앙상블 결과가 BOM size에서 ±10% 이상 벗어나면 이상치 플래그.

### 3. 비-베어링 필터 전략

![비베어링 도면에 K 실행 결과](/images/experiments/codex_nonbearing.jpg)

SPRING WASHER 도면에 K가 원을 잡았지만 OD/ID/W로서 의미가 없다. BEARING 키워드 부재 → soft filter로 자동 제외.

![Soft Filter penalty 누적](/images/experiments/codex_soft_filter.jpg)

87개 도면 중 BEARING ASSY는 **~7개**뿐. 나머지는 부품 도면(CASING, RING, PAD, PIN, WASHER 등)으로 OD/ID/W 개념 자체가 상이하다.

Codex 권장:

| 접근 | 설명 | 채택 |
|------|------|:---:|
| **점수 기반 (soft)** | confidence penalty 누적 → 0.6 미만은 비-베어링 | ✅ v5 적용 |
| 하드 리젝트 | 극단 케이스만 (OD < 30mm 등) | 부분 적용 |
| 제목 키워드 | BEARING/BRG 없으면 -15% | ✅ v5 적용 |

soft filter의 penalty 누적:
```
BEARING 키워드 없음 → -15%
OD 범위(50~2000mm) 밖 → -20%
OD ≤ ID → -25%
최악: 1.0 × 0.85 × 0.80 × 0.75 = 0.51
```

### 4. 87개 도면 실제 구성

![87개 도면 3종 비교](/images/experiments/codex_3types.jpg)

좌: ASSY 베어링(녹색, 7개) — OD/ID/W 적용 가능. 중: 단품 RING(주황, 50개) — 부분 적용. 우: 비베어링 WASHER(빨간, 25개) — OD/ID/W 무의미.

![87개 도면 구성 분포](/images/experiments/codex_87_composition.jpg)

| 카테고리 | 도면 수 | OD/ID/W 적용 | 비고 |
|---------|:------:|:----------:|------|
| ASSY (조립 베어링) | ~7 | ✅ 적합 | GT 가능 |
| 단품 (RING, PAD 등) | ~50 | △ 부분 적합 | GT 없음 |
| 비-베어링 (PIN, WASHER 등) | ~25 | ❌ 부적합 | OD/ID/W 무의미 |
| 완전 실패 | ~5 | — | OCR 자체 실패 |

**시사점**: 71% All-3 중 비-베어링 25개를 제외하면 실질 대상은 ~62개. 이 중 52개가 All-3 → 실질 검출률은 **52/62 = 84%**.

## 전체 기법 통합 — T1 도면

![T1 SECTION 전체 기법 오버레이](/images/experiments/combined_section_t1.jpg)

K(파란 원, 케이싱 565 ✗) + S01(빨간 화살촉, W=200 ✓) + S02(녹색, Ø360 ✓) + SEC(주황 스캔, OD=360 ID=190 W=200 ✓). 여러 기법이 각자의 강점으로 K의 약점을 보완하여 최종 3/3 달성.

## 권장 후속 작업

| 우선순위 | 작업 | 비고 |
|---------|------|------|
| 1 | monkey-patch 의존 제거 | S01/S02 독립 동작 보장 |
| 2 | 나머지 5개 ASSY GT 추출 | n=2 → n=7로 확대 |
| 3 | BOM size ±10% 자동 검증 | 이상치 조기 탐지 |
| 4 | 비-베어링 자동 분류 강화 | 도면 제목 + 형상 분석 |
