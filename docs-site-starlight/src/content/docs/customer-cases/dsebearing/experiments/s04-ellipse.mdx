> 부착 부품이 원 윤곽을 끊는 ASSY 도면에서 호 세그먼트를 그룹핑하여 완전 원을 복원한다.

## 핵심 아이디어

베어링 ASSY 도면에서 클램프, 브래킷 등이 원 윤곽을 끊는다.
`cv2.findContours`는 끊긴 윤곽을 별개 컨투어로 인식 → 원 피팅 실패.

이 방법은:
1. 컨투어를 **곡률**로 분석 → 호(arc) vs 직선(line) 세그먼트로 분해
2. 호 세그먼트의 곡률 반지름과 중심을 추정
3. 유사 반지름+중심의 호를 그룹핑 → **완전 원 복원**

## 파이프라인

| 단계 | 처리 | 출력 |
|------|------|------|
| 1. 곡률 분석 | 슬라이딩 윈도우 곡률 계산 | 세그먼트 분할점 |
| 2. 분류 | 평균 곡률 > threshold → 호, ≈0 → 직선 | 호/직선 리스트 |
| 3. 원 피팅 | 각 호에 원 피팅 → (cx, cy, r, arc_angle) | 호별 원 파라미터 |
| 4. 그룹핑 | 중심 거리 ≤15%, 반지름 차 ≤10% → 같은 원 | 복원된 원 리스트 |

### 필터 조건

- `r_min`: 최소 반지름 필터 (너무 작은 호 제거)
- `arc_angle`: 호 커버 각도 30° 이상
- 그룹 내 총 커버 각도로 최종 원 피팅 (Kasa method)

## 결과

### 원 검출 수

| 도면 | Decomp 결과 | RHT (S03) | 비고 |
|------|:----------:|:---------:|------|
| TD0062037 | **46** | 18 | 중간 수준 |
| TD0062042 | **41** | 20 | 중간 수준 |
| TD0062019 | **33** | 8 | 중간 수준 |

RANSAC(수백~수천)보다 훨씬 정제, RHT(8~20)보다는 많다.
**끊긴 원을 복원하는 데 특화**되어 있어, 원 후보 풀을 넓히는 역할.

### OD/ID/W 직접 기여

S03과 마찬가지로 독립적 OD/ID/W 분류는 하지 않는다. K방법 원 검출 보조 역할.

## 기존 cluster_arcs_to_circles와의 차이

| 항목 | 기존 (circle_ransac.py) | S04 (Ellipse Decomp) |
|------|----------------------|---------------------|
| 분해 단위 | fitEllipse (전체 컨투어) | **곡률 기반 세그먼트** |
| 그룹핑 | greedy merge | **중심+반지름 유사도** |
| 부분 호 | aspect ratio 필터 | **arc_angle 필터** |

## 한계

| 한계 | 설명 |
|------|------|
| 곡률 해상도 | 컨투어 포인트가 적으면 곡률 부정확 |
| 타원 오분류 | 단면도의 타원을 원으로 오인식할 수 있음 |
| 임계값 튜닝 | 곡률 threshold, 그룹핑 tolerance 조정 필요 |
