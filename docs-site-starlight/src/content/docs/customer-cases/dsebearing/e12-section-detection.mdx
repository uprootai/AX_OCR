# E12 — SECTION 단면도 감지 실험

> **목적**: K geometry가 정면도 케이싱 원을 잡는 문제를 SECTION 단면도 영역 직접 OCR로 우회
> **실험일**: 2026-03-30
> **결론**: SECTION 영역에서 핵심 치수(외경/내경/폭) 발견 성공. 단순 크기 기반 분류는 실패 — 위치 기반 분류 필요.

---

## 접근법

ASSY 도면은 일관된 레이아웃을 가짐:
- **좌측**: 정면도 (원형 단면 — 케이싱 외곽)
- **우측**: SECTION E-3 단면도 (직사각형 — 베어링 내부 구조)
- **상단**: 측면도 (축 방향 치수)

K geometry는 정면도의 원을 잡아 케이싱 외경을 반환. 실제 베어링 외경/내경은 SECTION 단면도에만 존재.

**현재 방법**: OCR로 "SECTION E-3", "GEN SIDE", "TBN SIDE" 텍스트를 찾고, 콘텐츠 밀도 스캔으로 도면 경계를 자동 감지. **하드코딩 없음** — 자동 감지 실패 시 SEC 메서드를 스킵한다.

![SECTION 자동 감지 — 하드코딩 vs OCR 앵커](/images/experiments/auto_detect_section.jpg)

빨간 점선 = 하드코딩(과하게 넓음). 녹색 실선 = OCR 앵커 + 콘텐츠 밀도로 자동 감지한 영역. 시안 점 = OCR 텍스트 앵커 위치.

## 핵심 발견

### 긍정적
1. **GT 값이 OCR에서 발견됨** — 190(T1/T2), 260(T3/T4/T5/T8), 200(T1/T2/T8), 420(T4), 48.5(Thrust)
2. **SECTION E-3 텍스트 자체도 OCR로 잡힘** — 영역 감지 자동화 가능
3. **크롭 영역이 일관적** — 7건 모두 같은 비율(53%~78%)로 SECTION 영역 커버

### 부정적
1. **단순 "가장 큰 값 = 외경" 분류 실패** — 케이싱 높이(700, 735) 등이 외경보다 큼
2. **Ø 기호가 잘 안 잡힘** — SECTION 단면도의 Ø380 같은 값이 `(0420)` 등으로 잡힘
3. **외경 치수가 세로 방향(우측)에 있음** — 위치 기반 분류 필요

## 분류 개선 방향

### SECTION 단면도의 치수 배치 패턴

```
         (폭)
    ┌────────────┐
    │            │ Ø외경 (우측, 세로 방향)
    │   (내경)   │
    │            │
    └────────────┘
         (하단 폭)
```

- **외경**: Ø 기호 + 우측 세로 방향 치수 (x좌표가 큰 영역)
- **내경**: 가운데 가로 방향 치수 (외경보다 작음)
- **폭**: 상단/하단 가로 방향 치수

### 구현 계획
1. 크롭 영역 내 텍스트의 **x/y 좌표** 활용
2. **우측 영역(x > 70%)의 세로 텍스트** → 외경 후보
3. **중앙 영역의 가로 텍스트** → 내경 후보
4. **상단/하단의 가로 텍스트** → 폭 후보
5. Ø 기호 우선, 괄호 치수 참고

## 코드

### SECTION 영역 자동 감지 (`_auto_detect_section`)
```python
# 1. 전체 이미지 PaddleOCR → 앵커 텍스트 찾기
for d in dets:
    text = d.get("text", "").upper()
    if "SECTION" in text and len(text) > 5:
        anchors["section"] = {"cx": cx, "cy": cy, "y_top": min(ys)}
    elif "GEN" in text and "SIDE" in text:
        anchors["gen"] = {"cx": cx, "cy": cy}
    elif "TBN" in text and "SIDE" in text:
        anchors["tbn"] = {"cx": cx, "cy": cy}

# 2. 좌우 경계: GEN SIDE ~ TBN SIDE (없으면 SECTION 기준 ±15%)
col_left = int(min(gen_x, tbn_x))
col_right = int(max(gen_x, tbn_x))

# 3. 상단 경계: 콘텐츠 밀도 스캔 (이진화 행별 밀도 < 0.005가 20행 연속 → 경계)
for row_y in range(label_y, 0, -1):
    density = np.sum(row > 0) / max(len(row), 1)
    if density < 0.005 and gap > 20:
        content_top = row_y + 20
        break

# 4. 마진 추가 (x 3%, y 2%) 후 비율 반환
return {"left": ..., "top": ..., "right": ..., "bottom": ...}
```

### 자동 감지 실패 시 (하드코딩 금지)
```python
sec_region = _auto_detect_section(image_path, _paddle, _img_bytes)
if not sec_region:
    # 하드코딩 폴백 없음 — SEC 메서드 스킵
    return {"method_id": "SEC", "od": None, "id_val": None, "width": None, ...}
```

### 세로 슬라이딩 스캔 (자동 감지 영역 기준)
```python
# 자동 감지된 SECTION 영역 내부의 상대 비율로 스캔
scan_left = sx1 + int((sx2 - sx1) * 0.30)   # SECTION 내부 30%
scan_right = sx1 + int((sx2 - sx1) * 0.80)  # SECTION 내부 80%
for frac in [0.15, 0.30, 0.45, 0.60, 0.75]:
    strip = img.crop((scan_left, scan_top, scan_right, scan_bot))
    rotated = strip.rotate(-90, expand=True)       # 세로 → 가로
    upscaled = rotated.resize((w*3, h*3))           # 3배 확대
    enhanced = Contrast(upscaled).enhance(1.8)      # 대비 강화
    detections = paddle_ocr(enhanced)
```

## v3: 세로 텍스트 슬라이딩 스캔 (2026-03-30)

### 핵심 발견

SECTION E-3 우측의 **세로 방향 Ø치수**(예: `Ø360.48 +0.03/0`)가 기존 OCR에서 감지되지 않던 원인:
1. 텍스트가 **세로 방향**(90° 회전)으로 표기
2. **도면 선/해칭과 겹침** → OCR 텍스트 감지 실패
3. **Ø 기호**가 OCR에서 누락 (PaddleOCR가 `0360` 또는 `(360.48`으로 인식)

### 해결 방법

**슬라이딩 윈도우 세로 텍스트 스캔**:

```python
# SECTION 우측 세로 영역을 15% 간격으로 5개 윈도우 스캔
for y_start in [0.28, 0.35, 0.42, 0.49, 0.56]:
    strip = img.crop((w*0.60, h*y_start, w*0.73, h*(y_start+0.15)))
    rotated = strip.rotate(-90, expand=True)     # 세로 → 가로
    upscaled = rotated.resize((w*3, h*3))         # 3배 확대
    enhanced = Contrast(upscaled).enhance(1.8)    # 대비 강화
    detections = paddle_ocr(enhanced)             # OCR
```

각 윈도우에서 `(\d{3,4}\.?\d*)` 패턴 추출 → Ø 접두사(`0360` → 360) 제거 → 세션명 힌트와 교차검증.

### 성공 요인 분석

1. **세로 텍스트 90° 회전** → 기존 OCR이 못 잡던 Ø치수를 PaddleOCR가 감지
2. **슬라이딩 윈도우** → 도면마다 다른 Ø치수 위치를 커버
3. **3배 확대 + 대비 강화** → 작은 텍스트도 OCR 감지 가능
4. **세션명 힌트 교차검증** → 여러 후보 중 정답 선택 (5/7 경우 힌트가 정확)

### W(폭) 개선: 양쪽 공통값 전략

W는 SECTION 상단(y가 20% 미만)과 하단(y가 75% 초과)에 동일 값으로 표기됨. **양쪽 공통값 = W** 전략:

```python
w_top = set(int(n) for n in top_nums)   # y < 0.20
w_bot = set(int(n) for n in bot_nums)   # y > 0.75
w_common = w_top & w_bot                 # 교집합
w_val = min(w_common) if w_common else max(w_bot)  # 폴백: 하단 최대
```

결과: W **0/5 → 5/7 (71%)**. T1/T2/T5/T8: 공통값 200 ✓, Thrust: 하단 48 ✓.

### 파이프라인 통합 + SEC-priority 전략

`dimension_ensemble.py`에 `_run_section_scan()` 함수로 통합:
- `run_ensemble()` 내 S06 뒤에 SEC 메서드로 참여
- ASSY 도면 자동 판별 (`drawing_title`에 "ASSY" 포함 시 실행)
- **SEC-priority 전략**: ASSY 도면에서는 K 대신 SEC 결과를 우선 사용
  - K는 ASSY 도면에서 케이싱 외곽을 잡는 경향 → OD 오분류
  - SEC는 SECTION 단면도에서 실제 베어링 치수를 직접 추출
