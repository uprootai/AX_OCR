## 미해결 질문

### BOM & 도면 관련

| # | 질문 | 배경 설명 | 왜 중요한가 | 상태 |
|---|------|----------|-----------|------|
| 1 | **T1, T2 BEARING ASSY 파싱 누락 — 수정 완료** | BOM PDF Page 2에서 프리앰블과 테이블이 합쳐지면서 T1(TD0062015), T2(TD0062021) 포함 64개 항목이 누락되었음. 파서 헤더 탐색 로직 수정으로 해결 → 326개 → **389개** (ASSY 9, SUB 19, PART 361) | 수정 후 T1~T8 + THRUST 9개 ASSY 정상 인식 | ✅ 수정 완료 |
| 2 | **395행 → 389개, 제외된 6개의 정체** | BOM Excel 원본 395행에서 6개를 빈 행·중복·참조 행으로 제외. 이 6개가 정확히 무엇인지 고객사 확인 필요 | 회의에서 "왜 395개가 아니라 389개냐"는 질문 대비 | ⏳ 미확인 |
| 3 | **87개 도면 중 단품도 정확 개수** | 87개 부품 도면에 조립도·단품도 혼재. *"1차 미팅에서 단품도 정확 개수는 확정되지 않았다"*고 기술 | 361개 PART에 대응하는 단품도가 87개 안에 몇 장 있는지 모르면 커버리지 계산 불가 | ⏳ 미확인 |
| 4 | **BOM 색깔 — 분홍 vs 노란이 실제로 구분되는지** | 1차 미팅 기록에는 "노란색"과 "하얀색" 2종류만 언급. 분홍색은 내부적으로 구분한 건지, Excel에 실제로 2가지 색이 있는 건지 | 만약 실제로 노란색 1종류라면 문서 전체의 🩷/🟨 구분이 잘못됨. **회의 전 BOM Excel을 열어서 직접 확인 필수** | ⏳ 미확인 |
| 5 | **동일 부품이 여러 ASSY에서 반복 사용되는지** | T5와 T6가 같은 사이즈(460x260). 동일 부품이 여러 어셈블리에 공용되면 견적을 1번만 내고 수량만 곱하면 됨 | 361개 PART 중 실제 고유 부품 수가 더 적을 수 있음. 중복 견적 방지 | ⏳ 미확인 |
| 6 | **도면 리비전 관리** | BOM REV 1 (2025.05.16, PT번호 변경/소재변경). 도면이 업데이트되면 이전 견적을 어떻게 갱신하는지 | 고객사의 리비전 관리 빈도·통보 방식에 따라 시스템 설계가 달라짐 | ⏳ 미확인 |

### 견적 프로세스 관련

| # | 질문 | 배경 설명 | 왜 중요한가 | 상태 |
|---|------|----------|-----------|------|
| 7 | **외주/자체제작 구분 기준** | 361개 PART 중 어떤 것이 외주이고 어떤 것이 자체 가공인지 | 외주 = 협력사 단가, 자체 = 내부 가공비로 견적 경로가 완전히 달라짐 | ⏳ 미확인 |
| 8 | **여유치 산정 기준 (몇 mm? 몇 %?)** | 고객 원문: *"큰 치수에 여유치를 붙여서"* — 고정값(+5mm)인지, 비율(+3%)인지, 부품 크기별로 다른지 | 여유치에 따라 소재 주문량이 달라지고 소재비가 직접 변동. 현재 시스템에 반영할 기준이 없음 | ⏳ 미확인 |
| 9 | **가공비 세부 기준** | 구멍 1개당 단가, 선반/밀링/드릴 등 가공 유형별 요율, 가공 난이도 분류 기준 | 현재 시스템은 부품 유형별 고정 단가(25종)로 추정 중. 실제 기준을 받아야 정밀화 가능 | ⏳ 미확인 |
| 10 | **체결부품(볼트·너트·와셔)의 견적 방식** | 15개 체결부품이 표준품(카탈로그 구매)인지, 주문 제작인지 | 표준품이면 가공비가 아니라 카탈로그 단가. 견적 방식 자체가 다름 | ⏳ 미확인 |
| 11 | **스크랩비·열처리비 등 비용 항목 범위** | 현재 시스템에 재료비+가공비+스크랩비+셋업비+검사비+운송비를 포함. 고객사 견적에도 이 항목들이 모두 포함되는지 | 견적서 양식 미수령으로 어떤 비용 항목을 넣어야 하는지 기준 없음 | ⏳ 미확인 |

| 기능 | 설명 | 상태 |
|------|------|------|
| **도면 파싱** | 타이틀블록·부품리스트·치수(17종 패턴) 자동 추출 | ✅ 테스트 완료 |

| **단가 관리** | 재질 21종, 가공비 25종, 수량 할인 4단계 | ✅ 테스트 완료 |
| **고객 프로필** | 고객별 할인율 적용 (DSE: 재료 5%, 가공 3%) | ✅ 테스트 완료 |

### 데이터 보안 정책

| 항목 | 정책 | 상태 |
|------|------|------|
| **도면 저장 위치** | 업루트 내부 서버(Docker 볼륨)에만 저장. 외부 클라우드 업로드 금지. | ✅ 적용 중 |
| **접근 권한** | 프로젝트 담당 개발자만 접근. 도면 경로는 `.gitignore`에 등록. | ✅ 적용 중 |
| **외부 API 전송** | 도면 이미지를 외부 LLM API(OpenAI, Claude 등)에 전송하지 않습니다. OCR/YOLO는 전부 **로컬 모델**. | ✅ 적용 중 |
| **분석 완료 후 삭제** | 업로드 파일 24시간 TTL 자동 삭제 적용. 결과 데이터 7일 보관 후 자동 정리. | ✅ 적용 중 |
| **시연 URL 보안** | ngrok 터널은 시연 시에만 활성화. 시연 종료 후 즉시 종료. | ✅ 적용 중 |
| **Git 리포 접근** | GitHub private 리포. 외부 공개 없음. | ✅ 적용 중 |

### 향후 배포 시 보안 옵션

| 옵션 | 설명 | 적합 대상 |
|------|------|---------|
| **On-Premise** | 내부 서버에 Docker 직접 설치. 도면이 외부로 나가지 않습니다. | 보안 요구 높은 고객 |
| **VPN 전용 클라우드** | 업루트 관리 서버 + VPN 접속. 인터넷 노출 없습니다. | 중간 보안 요구 |
| **SaaS** | 클라우드 호스팅 + TLS + 접근 제어. 빠른 시작. | 보안 요구 낮은 고객 |

:::tip[2차 미팅 시 전달 포인트]
"모든 도면 분석은 **로컬 AI 모델**로 처리합니다. 외부 서버로 도면을 보내지 않습니다. 납품 시에는 사내 서버에 직접 설치하는 On-Premise 옵션도 제공합니다."
:::
