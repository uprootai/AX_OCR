## SN 자동 검증 기능 반영

현재 파이프라인은 PDF 각 페이지의 `Doc No.`에서 SN(호선번호)을 정규식으로 추출합니다.
GAR(배치도), VD(밸브 상세도), ITDRA(계장기기도) 섹션으로 묶어 대표 SN을 계산합니다.
섹션 불일치가 있으면 `SN Validation` 시트에 남기고 VD↔ERP 매칭을 자동으로 멈춥니다.

<Aside type="note" title="현재 자동 판정">
예상 SN은 파일명 기준 `SN2709`입니다.
자동 검증 결과는 `GAR=SN2709`, `VD=SN2693`, `ITDRA=SN2709`입니다.
따라서 GVU VD 매칭은 자동 스킵 상태입니다.
</Aside>

### VD 표지 (Page 4): SN2693 — 불일치

![VD 표지 SN2693](/images/bmt/vd-cover-sn2693.png)

<Aside type="danger" title="CRITICAL: VD 도면 시리얼 불일치">
GAR 표지: **Doc No.: W5XGVU-SN2709-GAR**
VD 표지: **Doc No.: W5XGVU-SN2693-VD**

같은 PDF 파일 안에 두 개의 다른 시리얼 번호(= 서로 다른 배)가 존재합니다.
VD 섹션(10페이지, 밸브 8개)이 **다른 호선(SN2693)의 도면**으로 잘못 포함된 것으로 확인.
**SN2709용 VD 도면 재요청 완료 (2026-03-24)**.

**영향 범위**: 이 불일치로 막힌 것은 스키드 검증 3단계 중 **2단계(VD 하위 부품 비교)뿐**입니다.
1단계(TAG→Part List→ERP)와 3단계(ITDRA)는 모두 SN2709로 정상 테스트 가능합니다.
또한 2단계 비교 로직은 이미 FB2FB6(단품 밸브)에서 검증 완료된 것과 동일합니다.
</Aside>

## 도면 이미지 검출 한계 — YOLO 적용 불가

### 밸브 조립 단면도의 구조적 한계

![VD-02 조립도면](/images/bmt/vd02-full.png)

밸브 도면은 **어셈블리(조립) 상태 단면도**로 그려져 있습니다.

- 개별 부품이 결합되어 시각적으로 분리 불가
- 예: VD-01(Page 6) 30개 부품 중 29번(Upper Stem Bearing) 등이 단면도에서 보이지 않음
- 파나시아(P&ID)의 명확한 심볼과 달리, **YOLO 기반 심볼 검출 적용 불가**
- 통화 합의: 도면 이미지가 아닌 **도면 내 테이블(Description 표)**을 기준으로 검증

<Aside type="note" title="검증 기준">
도면 우측의 Description 테이블(NO./DESCRIPTION/MATERIAL/Q'TY)이 기준(정답)입니다.
도면 이미지(기계 단면도)는 보조 참조용으로만 사용합니다.
`pdfplumber`로 VD 8페이지 전체 테이블 추출 성공 확인 (151개 부품 (raw 143행, 병합셀 split 후 151개)).
ITDRA도 `pdfplumber`로 7페이지 전부 추출 성공했습니다. `PyMuPDF`는 SN과 Balloon 보조 분석에 사용합니다.
</Aside>

## 현재 진행 가능한 범위

| 항목 | 가능 여부 | 근거 |
|------|----------|------|
| 1번 자료(단품 밸브) 자동 비교 | ✅ 즉시 가능 | 27개 부품 추출 + ERP 매칭 완료 |
| 2번 자료 VD 테이블 추출 | ✅ 기술적으로 가능 | 8페이지 151개 부품 (raw 143행 + 병합셀 split 8개) pdfplumber 추출 성공 |
| 2번 자료 VD↔ERP BOM 매칭 | 🔴 자동 스킵 | SN 검증 결과 VD=SN2693, ERP=SN2709 |
| 2번 자료 ITDRA 테이블 추출 | ✅ 기술적으로 가능 | 7페이지 `Subject/Description`, `POS/DESCRIPTION` pdfplumber 추출 성공 |
| 2번 자료 GAR 배치도 분석 | ⚠️ 테이블 아닌 도면 | 텍스트 추출은 가능하나 구조화 필요 |
| fuzzy 매칭 보강 (T1↔T4) | ✅ 구현 가능 | Levenshtein 거리 기반 |
| prefix 매칭 (-KGS, -LIP) | ✅ 구현 완료 | 5건 자동 검출 확인 |
| 동의어 사전 (ANSI↔HB) | ✅ 구현 완료 | 코드 체계 상이 케이스 자동 연결 |

## 경쟁 상황

- 다른 개발업체가 동일 건으로 이미 테스트 완료 및 제안 진행 중
- BMT 측 이사: "OCR로 읽는 거 어렵지 않다"는 인식 (다른 업체 테스트 경험 기반)
- BOM에 있고 도면에 없는 정보 → 추후 엑셀로 리스트업 하겠다고 함
