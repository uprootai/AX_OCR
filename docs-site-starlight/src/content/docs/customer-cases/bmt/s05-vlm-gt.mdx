> GAR 배치도를 6개 영역으로 크롭하여 Claude Vision(멀티모달)으로 직접 TAG를 확인, **24개 TAG Ground Truth**를 확립한 결과입니다.

---

## 왜 필요한가

S01 벤치마크에서 사용한 GT(정답)는 Part List 엑셀에서 추출한 28개 TAG였습니다.
하지만 **실제 이미지에 표기된 TAG**와 엑셀 목록이 다를 수 있어, 직접 이미지를 보고 확인이 필요했습니다.

---

## 분석 방법

1. GAR 3페이지를 4x 해상도(3368×4764px)로 추출
2. 6개 영역으로 크롭
3. 각 크롭 이미지를 Claude Vision으로 직접 분석
4. TAG와 치수를 구분하여 기록

---

## OCR vs Claude Vision 비교

| 방식 | Recall | Precision | 비용 | 속도 |
|------|--------|-----------|------|------|
| PaddleOCR 전체 (S01) | 41.7% | - | 무료 | 5.4s |
| PaddleOCR 뷰별 (S02) | 95.8% | 100% | 무료 | 11.7s (합계) |
| **Claude Vision (크롭)** | **100%** | **100%** | API 비용 | ~30s |

<Aside type="note" title="실용적 결론">
Claude Vision은 GT 확립(정답 만들기)에 유용하지만, 실서비스에서는 비용/속도 면에서 PaddleOCR + 뷰 분리(95.8%)가 현실적입니다.
하이브리드: PaddleOCR 1차 → 신뢰도 낮은 결과만 Claude Vision으로 2차 검증하는 방식도 가능합니다.
</Aside>
