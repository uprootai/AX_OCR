## S02: 빈 공간 프로파일 기반 경계 감지

뷰 라벨 중심점 + 이미지 밝기 프로파일에서 빈 수평/수직 영역을 찾아 경계선을 자동 계산합니다.

### 초기 버전 (2행×3열 그리드)

![초기 그리드 분리](/images/bmt/e11-s02-auto-sections.png)

문제: 하단 3D 투시도(TAG 밀집 영역)가 그리드 밖에 있어서 도면이 잘림.

### 개선 버전 (6개 그리드 + 하단 보충)

![개선된 영역 분리](/images/bmt/e11-v3-sections.png)

8개 영역으로 분리: 상단 6개 뷰 + **하단 투시도 1개** + title block 1개.

![하단 투시도 크롭](/images/bmt/e11-v3-lower-full.png)

`lower_full` 크롭에 핵심 TAG 14개(V09, V11-1/2, V15, V04, B02, PT05, V07, V14, B01, V02, V03, V05, PT03, PT07)가 모두 포함됩니다.

**경계 계산 (하드코딩 0 — 빈 공간 프로파일 자동 감지):**

```
수평 경계 (이미지 행별 평균 밝기 > 248인 연속 영역):
  y=[718~757] → 상단뷰 / 중단뷰 분리
  y=[1295~1314] → 중단뷰 / 하단 투시도 분리
  y=[2246~2263] → 투시도 / title block 분리

수직 경계 (이미지 열별 평균 밝기 > 248인 연속 영역):
  x=[630~652] → 좌/중 열 분리
  x=[1143~1172] → 중/우 열 분리
```

경계 근처 TAG를 놓치지 않도록 60px 오버랩을 추가한 크롭도 함께 실행합니다.

|------|---------|------|
| 기본 크롭 | 6개 | 행/열 경계로 정확히 분리 |
| 오버랩 크롭 | 4개 | 경계 ±60px 확장 (경계 근처 TAG 보완) |

## S04: OCR 앙상블

각 크롭에 PaddleOCR + Tesseract를 모두 실행하고 결과를 합산합니다.

| 엔진 | 강점 | E10에서 확인된 보완 |
|------|------|-------------------|
| PaddleOCR | 범용 TAG 인식 | 대부분의 TAG 검출 |
| Tesseract | 회전/노이즈 텍스트 | V11-2, V15 등 PaddleOCR이 놓친 TAG |

합산 결과: 기본 크롭(21/24) + 오버랩 크롭 + 앙상블 = **24/24 = 100%**

<Aside type="tip" title="수동보다 좋은 결과">
E10 수동 크롭(95.8%)은 PI02 1개를 놓쳤지만, E11 자동 파이프라인은 **24개 전부 검출**했습니다.
오버랩 + 앙상블 조합이 수동 크롭의 한계를 넘어선 것입니다.
</Aside>

## VLM 콘텐츠 기반 크롭 (멀티모달 LLM)

Claude Vision이 이미지를 직접 보고 "도면 콘텐츠 영역"을 판단한 결과입니다.
직선 그리드가 아닌, 실제 도면 콘텐츠 기준 비정형 영역.

![VLM 판단 영역](/images/bmt/e11-vlm-regions.png)

### VLM 영역의 효율성

| 영역 | TAG 수 | 누적 Recall |
|------|--------|-----------|
| `vlm_front_dims` (FRONT VIEW 치수) | 6개 | 25% |
| **`vlm_front_3d` (3D 투시도)** | **16개** | **91.7%** |
| + 빈공간 보충 (gap_mid_left/center) | 2개 | **100%** |

<Aside type="tip" title="VLM 1개 영역 = 67% TAG">
VLM이 "핵심 콘텐츠"로 판단한 `front_view_3d` 영역 **1개만으로 16/24 TAG**를 검출합니다.
이것은 직선 그리드 방식으로는 불가능 — 이 영역이 여러 그리드 셀에 걸쳐 있기 때문입니다.
</Aside>

### 최적 전략: VLM + 빈공간 하이브리드

1. **VLM** → 핵심 콘텐츠 영역 2~3개 판단 (TAG 대부분 커버)
2. **빈공간 프로파일** → 나머지 영역 보충 크롭
3. **PaddleOCR 앙상블** → 모든 크롭에서 TAG 추출 + 합산

## 최종 파이프라인: 테이블 제거 + 뷰 라벨 분할 + 보충 크롭

### 시각화 — 7개 뷰 분리 (유동적 경계)

![뷰 분리 최종](/images/bmt/e11-split-organic.png)

blob #0(중단 3뷰 합쳐짐)을 뷰 라벨 위치 기준으로 BOTTOM VIEW BOLTING / RIGHT VIEW / A VIEW로 재분할.

### 테이블 제거 결과

![테이블 제거 후 blob](/images/bmt/e11-no-table-blobs.png)

우측 GVU SPEC + EXTERNAL PIPE 테이블을 감지하여 제거. 도면 뷰만 남김.

### 최종 결과

| 단계 | 방법 | Recall |
|------|------|--------|
| 테이블 제거 + 5 blob | hull 마스킹 | 87.5% |
| blob 내부 뷰 라벨 재분할 | hull 마스킹 | 87.5% |
| **뷰 분할 + bbox 크롭 (마스킹 제거)** | **bbox만** | **95.8%** |
| **+ 보충 크롭 (lower_full)** | **bbox + 보충** | **100%** |

<Aside type="danger" title="hull 마스킹은 해로움">
convex hull로 영역 밖을 흰색 처리하면 경계 근처 TAG(B01, B03, V08)가 잘립니다.
크롭은 bbox(직사각형)로, 유동적 경계는 시각화 용도로만 사용해야 합니다.
</Aside>

## 최종 결과: 12영역 + 오버랩 80px

### 전체 뷰 맵 (오버랩 적용)

![최종 뷰 분리](/images/bmt/e11-overlap-views.png)

각 뷰 크롭에 경계 ±80px 오버랩을 적용하여 도면 잘림을 방지합니다.

### 뷰별 크롭 품질

| 뷰 | 이전 (잘림) | 오버랩 후 |
|----|-----------|---------|
| BOTTOM VIEW | GVU 본체 우측 잘림 | ✅ 전체 포함 |
| RIGHT VIEW | A VIEW 도면 잘림 | ✅ 전체 포함 |
| A VIEW | 하단 잘림 | ✅ 단면도 전체 포함 |

### 전체 파이프라인 (하드코딩 0)

```
1. PaddleOCR → 뷰 라벨 6개 자동 검출
2. 라벨 cy → 행 클러스터 (2행)
3. 라벨 cx → 열 클러스터 (3열)
4. 인접 라벨 중간점 → 빈공간 프로파일 → 경계 정교화
5. 2×3 그리드 = 6셀 (각 ±80px 오버랩)
6. 하단 보충 3개 + 우측 테이블 2개 = 총 12영역
7. PaddleOCR + Tesseract 앙상블 + fuzzy 교정(Z→7)
→ 24/24 = 100% TAG Recall
```
