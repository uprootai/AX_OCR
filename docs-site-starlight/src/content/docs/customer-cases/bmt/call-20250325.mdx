## 핵심 결론: 우리가 잘못 이해하고 있었다

<Aside type="danger" title="근본적 전환">
우리는 **VD 상세도면의 테이블을 추출**해서 ERP BOM과 재질/수량을 비교하는 것이 핵심이라고 판단했습니다.

실제로는 **GAR 배치도(3페이지) 이미지에서 TAG를 추출**하고, Part List를 거쳐 **BOM에 해당 코드가 있는지 누락 여부**를 확인하는 것이 핵심입니다.
</Aside>

### 박순근 팀장님 설명 요약

```
"세 번째 장이 핵심이에요."
  → GAR 배치도 (Top View, Front View 등)

"여기 PT07, TT01, V15, V04 이런 거 보이세요? 걔네들이 각 부속들이에요."
  → 이미지에 TAG 라벨이 흩어져 있음

  → BOM 존재 여부 확인이 최종 목표

  → 재질/치수 비교 불필요, 코드 매칭만
```

## R&D 판단 전환

<Aside type="tip" title="R&D가 오히려 필요해졌다">
**이전 판단**: 벡터 PDF → pdfplumber 테이블 추출 → AI 불필요 → 시스템 엔지니어링

**통화 후 판단**: GAR 배치도 = 복잡한 엔지니어링 이미지
- TAG가 이미지 곳곳에 흩어져 있음 (Top View, Front View, Right View 등)
- 텍스트가 회전되어 있거나 겹칠 수 있음
- 단순 테이블 추출이 아닌 **이미지 내 텍스트 인식** 필요
- pdfplumber로는 안 됨 → **OCR / 이미지 AI 기술 필요**
- **AI 바우처 R&D 정당화 가능!**
</Aside>

### 기술 요구사항 변화

| 항목 | 이전 | 통화 후 |
|------|------|--------|
| PDF 처리 | pdfplumber (테이블 추출) | **이미지 OCR + 좌표 기반 텍스트 추출** |
| 비교 로직 | exact + prefix + fuzzy + synonym | **TAG 존재 확인 (단순 매칭)** |
| AI 모델 | 불필요 | **이미지 내 텍스트 인식 모델 필요** |
| 난이도 | 문자열 매칭 (쉬움) | **엔지니어링 도면 레이아웃 이해 (어려움)** |
| R&D 가치 | 낮음 | **높음** — 다양한 도면 레이아웃 대응 필요 |

## 이전 분석 중 여전히 유효한 것

| 항목 | 유효 여부 | 비고 |
|------|----------|------|
| SN 자동 검증 (SN2709 vs SN2693) | ⚠️ 부분 유효 | SN 검증 자체는 유용하나, VD 비교가 핵심이 아니므로 우선순위 하락 |
| Part List 엑셀 파싱 | ✅ 유효 | TAG → 품목코드 변환에 필수 |
| ERP BOM 엑셀 파싱 | ✅ 유효 | 품목코드 존재 확인에 필수 |
| Balloon 교차검증 | ⚠️ 부분 유효 | GAR 배치도의 TAG 추출과 연관될 수 있음 |
| FB2FB6 POC 결과 (27개 매칭) | ❌ 핵심 아님 | 단품 밸브는 별개 제품. 참고 자료로만 유지 |

| 동의어 사전 (ANSI↔HB) | ❌ 불필요 | 코드 존재 여부만 확인하므로 |
