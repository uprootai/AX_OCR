## 노드 상세

### ① ImageInput — GAR 배치도 입력

입력: GAR 배치도 이미지 (래스터 PNG/JPG)

### ② View Splitter — 5개 뷰 영역 분리

S02에서 검증된 크롭 좌표 사용:

| 뷰 | 좌표 (비율) | 검출 TAG 수 |
|----|-----------|------------|
| Bottom View | `[0, 0, 0.50, 0.45]` | 5개 |
| Front View 상단 | `[0.40, 0, 1.0, 0.40]` | 2개 |
| Front View 하단 | `[0.10, 0.62, 0.65, 0.88]` | 9개 |
| Bottom 중앙 | `[0.30, 0.75, 0.75, 0.98]` | 5개 |
| Right/A View | `[0.30, 0.30, 1.0, 0.70]` | 2개 |

### ③ PaddleOCR — 뷰별 OCR

각 크롭에 PaddleOCR(영어 모드) 실행. S01에서 7개 엔진 중 1위.

### ④ TAG Filter — 알파벳 규칙

S03에서 검증된 필터링 규칙 적용. 39개 raw → 24개 TAG.

### ⑤ Excel Lookup — Part List 조회

Part List 엑셀에서 TAG → 품목코드 변환:
- VALVE LIST: B열(TAG) → Y열(ERP CODE)
- SENSOR LIST: B열(Instrument Code) → M열(품목코드)

### ⑥ BOM Check — ERP BOM 누락 확인

ERP BOM 엑셀의 C열(자품목)에 해당 코드가 존재하는지 검색.

### ⑦ Report — 누락 리포트

3개 시트 XLSX 출력: Summary / TAG Details / Mismatches

## 현재 한계 및 후속 작업

<Aside type="caution" title="executor 미구현">
현재 템플릿은 노드 배치와 파라미터만 정의되어 있습니다.
실제 실행하려면 gateway-api에 다음 executor가 필요합니다:

- `view_splitter_executor` — 이미지를 5개 뷰로 크롭
- `tag_filter_executor` — OCR 결과에서 TAG만 필터링
- `excel_lookup_executor` — Part List 엑셀 조회
- `bom_check_executor` — ERP BOM 존재 확인

이것은 후속 Epic(E11)에서 구현할 예정입니다.
</Aside>
