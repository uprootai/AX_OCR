> 수령한 자료 전체를 대상으로 자동 비교를 시도한 결과입니다. **자동화 수준을 정직하게 기록합니다.**

<Aside type="note" title="이 페이지에서 사용하는 용어">
- **FB2FB6** = 단품 밸브 모델명 (1번 자료)
- **Part List** = 설계 부서의 부품 목록 엑셀. VALVE LIST / SENSOR LIST / FITTING LIST / TECH SPEC 4개 시트
- **ERP BOM** = ERP 시스템의 공식 부품 목록 (자재 발주 기준)
- **VD (Valve Drawing)** = 밸브 상세도. 현재 SN2693(다른 호선)이라 ERP BOM과 비교 불가
- **ITDRA** = 계장기기도. 온도/압력센서 스펙 도면
- **TAG** = 스키드 내 장비 위치 식별자 (V01, PT01 등)
- **Drawing No.** = 도면 번호. TAG와 ERP BOM 코드를 연결하는 중간 키

상세한 파일 구조는 [고객 확인 현황 → 수령 자료 & 용어 정리](./customer-confirmations#수령-자료--용어-정리)를 참고하세요.
</Aside>

## 비교 1: FB2FB6 도면 vs ERP BOM — ✅ 완전 자동

**이 비교는 100% 자동입니다.** 사람 개입 없이 파이프라인이 돌아갑니다.

- 도면 PDF → `pdfplumber` 테이블 추출 (병합셀 `split('\n')` 후처리) → 27개 부품번호
- ERP BOM 엑셀 → `zipfile` + `xml.etree` 파싱 → 27개 자품목코드
- 다단계 매칭: exact → prefix → fuzzy → synonym

| 분류 | 건수 | 자동화 |
|------|------|--------|
| ✅ 완전 일치 (exact) | 18건 | 자동 |
| ⚠️ 접미사 상이 (prefix) | 5건 | 자동 |
| 🟡 fuzzy 매칭 (91~96%) | 3건 | 자동 |
| 🟣 동의어 사전 (ANSI↔HB) | 1건 | 자동 연결 |

[1:1 대조 상세 → POC 테스트 결과](./poc-results)

## 비교 2: Part List Fitting List vs ERP BOM — ✅ 완전 자동

피팅 파트리스트와 ERP BOM의 **코드가 동일 체계**여서 직접 비교 가능.

**결과: 27/27 매칭 (100%)**

| 구분 | 건수 | 예시 |
|------|------|------|
| 완전 일치 (100%) | 25건 | SPC-12M = SPC-12M |
| fuzzy 일치 (90%) | 2건 | SFC-16M-8G ↔ SGC-16M-8G |

<Aside type="note" title="왜 잘 되나?">
피팅 코드(SPC, SAE, SMC, SME 등)는 Superlok 사의 표준 코드 체계입니다.
Part List와 ERP BOM 모두 **같은 코드 체계**를 쓰기 때문에 문자열 비교만으로 100% 매칭됩니다.
유일한 차이는 SFC↔SGC 2건으로, fuzzy(90%)로 자동 검출됩니다.
</Aside>

## 비교 3: Part List VALVE LIST vs ERP BOM — ❌ 자동 매칭 불가

<Aside type="danger" title="이 비교는 자동이 아닙니다">
VALVE LIST의 TAG(V01, V03...)와 ERP BOM의 자품목코드(FB2F29-R64A-A111-36L-EXT-GVU-DNV)는
**코드 체계가 완전히 다릅니다.** TAG는 위치 식별자이고, ERP 코드는 제품 모델명입니다.
이 둘을 연결하려면 **VD 도면의 Drawing No.**가 필요합니다 (3단계 매칭: TAG → Drawing No. → ERP).
현재 VD 도면이 SN2693(다른 호선)이라 자동으로 이 매핑을 만들 수 없습니다.
</Aside>

### Part List VALVE LIST 내용 (18개)

| No | TAG | Type | Q'TY | Medium |
|----|-----|------|------|--------|
| 1 | V01 | Ball | 1 | CH4 |
| 2 | B01 | Filter | 1 | CH4 |
| 3 | V03 | DBB | 1 | CH4 |
| 4 | V05 | (V03과 동일) | - | - |
| 5 | V04 | Ball | 1 | CH4 |
| 6 | FT01 | Flow Meter | 1 | CH4 |
| 7 | V06 | Gas Control Valve | 1 | CH4 |
| 8 | V08 | Ball | 1 | N2 |
| 9 | V14 | DBB | 1 | N2 |
| 10 | V07 | (V14와 동일) | - | - |
| 11 | V15 | Ball | 1 | CH4 |
| 12 | V11-2 | Ball | 1 | N2 |
| 13 | V11-1 | Pan Check | 1 | N2/CH4 |
| 14 | V02 | Ball | 1 | CH4 |
| 15 | B03 | Filter | 1 | N2 |
| 16 | ORI | Orifice | 1 | N2 |
| 17 | V09 | Ball | 1 | Air |
| 18 | B02 | Air Filter Regulator | 1 | Air |

### 자동 매칭이 안 되는 이유

```
Part List:  TAG = "V01"     ← 위치 식별자
VD 도면:    Drawing No. = "FB2F29-R64A-A111-36L-EXT-GVU-DNV"   ← 제품 코드
ERP BOM:    자품목 = "FB2F29-R64A-A111-36L-EXT-GVU-DNV"        ← 제품 코드

V01 → FB2F29... 매핑은 VD 도면의 타이틀 블록에서 추출해야 함
BUT: VD 도면이 SN2693 (다른 호선) → 매핑 생성 불가
```

### SN2709 도면 수령 시 가능한 자동화

올바른 VD 도면(SN2709)을 수령하면:
1. VD 각 페이지의 타이틀 블록에서 TAG + Drawing No. 자동 추출 가능 (pdfplumber/PyMuPDF)
2. TAG → Drawing No. 매핑 테이블 자동 생성
3. Drawing No.로 ERP BOM과 exact match

**이 3단계가 구현되면 VALVE LIST 매칭이 완전 자동화됩니다.**

## 비교 4: Part List SENSOR LIST vs ERP BOM — ❌ 자동 매칭 불가

<Aside type="danger" title="이 비교도 자동이 아닙니다">
센서 TAG(PT01, TT01, GSO/GSC-V02)와 ERP BOM 코드(PT-MBS4201-4308G-DANFOSS, TT-MBT5252-9110-DANFOSS)는
**코드 체계가 완전히 다릅니다.** 문자열 비교로는 연결할 수 없습니다.
</Aside>

### SENSOR LIST 내용 (주요 항목)

| 구분 | 항목 | 건수 |
|------|------|------|
| Actuator | HKC社 HP-145S, HP-75S, HP-50S | 7개 |
| Solenoid Valve | CV-V02~V15 | 7개 |
| Limit Switch | GSO/GSC-V02~V15 | 7개 |
| Pressure Transmitter | PT01~PT07 | 7개 |
| Temp. Transmitter | TT01 | 1개 |
| Flow Meter | FT01 | 1개 |
| 기타 (Manifold, JB, PB, Spare) | - | 11개 |

### 매칭이 안 되는 이유

```
Part List:  TAG = "PT01"                          ← 위치 식별자
ITDRA 도면: Maker = "Danfoss", Model = "MBS 4201"  ← 제품 정보
ERP BOM:    코드 = "PT-MBS4201-4308G-DANFOSS"      ← Maker+Model 조합

PT01 → Danfoss MBS 4201 → PT-MBS4201-... 매핑은
ITDRA 도면의 스펙 테이블에서 TAG↔Maker/Model을 추출해야 함
```

### 해결 방향

ITDRA 도면에서 TAG↔Maker/Model 추출이 되면 (비교 6 참조):
1. ITDRA의 `Tag No. = PT01`, `Maker = Danfoss`, `Model = MBS 4201` 추출
2. ERP BOM에서 `PT-MBS4201` prefix 매칭
3. PT01 → PT-MBS4201-4308G-DANFOSS 자동 연결

**현재는 ITDRA 파싱이 불완전하여 이 파이프라인을 구현할 수 없습니다.**

## 비교 5: VD 도면 8페이지 추출 — ⚠️ 추출만 자동, 비교 불가

pdfplumber로 Description 테이블 추출은 성공. **SN2693(다른 호선)이라 ERP BOM과 비교는 의미 없음.**

| VD | Page | TAG | 부품 수 | 병합셀 | 추출 판정 |
|----|------|-----|---------|--------|---------|
| VD-01 | 6 | V01 | 30개 | 6개 복원 | ⚠️ 후처리 필요 |
| VD-02 | 7 | V03/V05 | 22개 | 2개 복원 | ⚠️ 후처리 필요 |
| VD-03 | 8 | V04/V15 | 14개 | 2개 복원 | ⚠️ 후처리 필요 |
| VD-04 | 9 | V08 | 17개 | 없음 | ✅ 완벽 |
| VD-05 | 10 | V07/V14 | 21개 | 4개 복원 | ⚠️ 후처리 필요 |
| VD-06 | 11 | V11-2 | 20개 | 없음 | ✅ 완벽 |
| VD-07 | 12 | V11-1 | 8개 | 없음 | ✅ 완벽 |
| VD-08 | 13 | V02 | 19개 | 2개 복원 | ⚠️ 후처리 필요 |
| **합계** | | | **151개** | **16개** | **8/8 추출 성공** |

<Aside type="note" title="SN2709 도면 수령 시">
올바른 VD 도면을 수령하면:
1. 각 VD 페이지의 타이틀 블록에서 TAG + Drawing No. 추출
2. Description 테이블의 부품 리스트 추출 (병합셀 후처리)
3. Drawing No.로 ERP BOM의 해당 밸브 하위 BOM과 비교
이 전체 과정이 자동화 가능합니다.
</Aside>

pdfplumber 실패 → PyMuPDF 텍스트 + 정규식 파싱.

| Page | Tag No. | Maker | 추출 정확도 |
|------|---------|-------|-----------|
| 16 | TT01 | Danfoss | ⚠️ Tag OK, Maker에 Model이 섞임 |
| 17 | PT05/PT06 | Danfoss | ⚠️ 동일 |
| 18 | PT01/PT03/PT04 | Danfoss | ⚠️ 동일 |
| 19 | PT07 | ANALYSIS Ltd | ⚠️ 동일 |
| 20 | GSO/GSC-V03/V05 | PEPPERL+FUCHS | ⚠️ 동일 |
| 21 | GSO/GSC-V02/V04/V15 | PEPPERL+FUCHS | ⚠️ 동일 |
| 22 | FT01 | Endress+Hauser | ⚠️ 동일 |

<Aside type="caution" title="ITDRA 파싱 한계">
- Tag No.는 추출 가능하나, 일부 페이지에서 파싱 실패 (Page 20, 21)
- Maker와 Model이 분리되지 않고 섞여서 나옴
- 부품 테이블(POS/DESCRIPTION)은 구조화 추출 실패
- **결론: ITDRA는 현재 자동화 신뢰도가 낮아서 수동 확인이 필요합니다**
</Aside>

## 비교 7: 표제란 SN2711 참조

Part List 엑셀 내부의 `표제란` 시트에서 **SN2711** 참조 발견.

| 항목 | 값 |
|------|---|
| Hull No. | ANDO2711/2/3/4 |
| Classification | LR (메인 시트는 DNV) |
| Doc. No | **W5XGVU-SN2711-PLRA** |

메인 Part List(VALVE LIST, SENSOR LIST 등)는 SN2709인데 표제란만 SN2711입니다.
다른 호선의 템플릿이 남아있는 것으로 추정. **고객 확인 필요.**

## 전체 요약 — 정직한 버전

| 비교 | 자동화 수준 | 매칭률 | 현실 |
|------|-----------|--------|------|
| FB2FB6 도면 vs ERP | **✅ 완전 자동** | 100% (27/27) | 병합셀 후처리 + fuzzy + 동의어 사전으로 동작 |
| Fitting List vs ERP | **✅ 완전 자동** | 100% (27/27) | 코드 체계가 같아서 쉬움 |
| VALVE LIST vs ERP | **❌ 수동 매핑 필요** | - | VD 도면(SN2709) 수령 후 자동화 가능 |
| SENSOR LIST vs ERP | **❌ 자동 불가** | - | ITDRA 파싱 완성 후 자동화 가능 |
| VD 도면 추출 | **⚠️ 추출만 자동** | - | 151개 부품 추출 OK, 비교 대상 없음 |
| ITDRA 추출 | **⚠️ 부분만** | - | Tag/Maker 추출 불완전 |

<Aside type="tip" title="핵심 결론">
**지금 자동으로 되는 것**: 코드 체계가 동일한 경우 (FB2FB6, Fitting) — exact + prefix + fuzzy로 90~100% 매칭.

**SN2709 도면 수령 후 자동화 가능한 것**: VALVE LIST 3단계 매칭 (TAG → Drawing No. → ERP).

**추가 개발 필요한 것**: ITDRA 구조화 파서 → SENSOR LIST 매칭 파이프라인.

**자동 연결된 것**: ANSI↔HB 같은 코드 체계 상이 케이스도 동의어 사전으로 처리합니다.
</Aside>
