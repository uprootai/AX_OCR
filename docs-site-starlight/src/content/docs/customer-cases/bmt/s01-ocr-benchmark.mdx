## 엔진별 분석

| Engine | 특성 | 이 이미지에서의 문제 |
|--------|------|-------------------|
| **PaddleOCR** | 범용 OCR, 영어 모드 | 가장 나은 결과, 하지만 42.9% |
| **Tesseract** | 전통 OCR | 노이즈 많음, 작은 텍스트 놓침 |
| **EasyOCR** | 한국어 기본 설정 | 영어 TAG를 한국어로 인식 시도 |
| **DocTR** | 2-stage (검출+인식) | 텍스트 영역 검출 자체가 실패 |
| **eDOCr2** | 치수 전문 OCR | 치수(54개)는 잘 잡으나 TAG에 부적합 |
| **Surya** | 레이아웃 분석 | 모듈 에러 (requests 미설치) |
| **TrOCR** | Transformer OCR | 단어 1개만 인식 (전체 이미지 미지원) |

<Aside type="caution" title="42.9%는 전체 이미지 기준">
전체 이미지(1684×2382px)를 한 번에 넣었을 때의 결과입니다.
뷰별 분리(S02)나 멀티모달 VLM(S05)으로 개선 가능성이 있습니다.

특히 **16개 미검출 TAG 중 상당수는 이미지의 작은 영역에 밀집**되어 있어,
해당 영역을 크롭하면 검출률이 올라갈 수 있습니다.
</Aside>
