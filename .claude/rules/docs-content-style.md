# Docs Content Style Rules

## 1. Frontmatter 필수 필드

모든 `.md` / `.mdx` 파일에 아래 4개 필드 필수:

```yaml
---
title: "문서 제목"
description: "한 줄 요약"
tags: ["태그1", "태그2"]       # 최소 2개
sidebar_position: 1             # 사이드바 순서
---
```

- `title`: 문서 제목
- `description`: 한 줄 요약 (검색/SEO용)
- `tags`: 분류 태그 배열 — 최소 2개 이상
- `sidebar_position`: 사이드바 정렬 순서 (숫자)

## 2. H1 제목 패턴

반드시 괄호 안에 번역을 포함:

- **한글 문서**: `# 한글 제목 (English Title)`
- **영어 문서**: `# English Title (한글 설명)`

예시:
```markdown
# 시스템 개요 (System Overview)
# YOLO 검출 API (YOLO Detection)
# Batch Delivery (일괄 납품)
```

## 3. Blockquote 요약

- H1 바로 다음 줄에 `> ` 시작하는 **1문장 요약** 필수
- 홍보성 표현 금지: "좋습니다", "강력합니다", "멋집니다", "최고의", "혁신적인"

```markdown
# 시스템 개요 (System Overview)
> 기계 도면 자동 분석 및 제조 견적 시스템의 전체 구조를 설명한다.
```

## 4. 섹션 순서 (3종 템플릿)

### Type A — Overview / Index 페이지

```
개요 → 아키텍처/구조 → 하위 페이지 → 관련 API → 관련 문서
```

### Type B — Page 상세 페이지

```
접속 방법 → 탭/구조 → 상세 설명 → 관련 API → 관련 문서
```

### Type C — API Reference 페이지

```
기본 정보 → 파라미터 → 응답 → 사용 예시 → 리소스 → 관련 문서
```

## 5. 표 형식 표준

용도별 표 헤더:

| 타입 | 용도 | 헤더 |
|------|------|------|
| Type A | URL/접근 | `\| URL \| 권한 \| 설명 \|` |
| Type B | 라우팅 | `\| 경로 \| 페이지 \| 설명 \| 가이드 \|` |
| Type C | 탭 구조 | `\| 탭 \| 설명 \| 주요 동작 \|` |
| Type D | 기능 목록 | `\| 기능 \| 설명 \|` |
| Type E | API 목록 | `\| 메서드 \| 엔드포인트 \| 설명 \|` |
| Type F | 파라미터 | `\| 파라미터 \| 타입 \| 기본값 \| 설명 \|` |
| Type G | 키-값 | `\| 항목 \| 값 \|` |

## 6. 관련 API 섹션

- `## 관련 문서` 바로 위에 `## 관련 API` 배치
- 표 형식: `| 메서드 | 엔드포인트 | 설명 |`

```markdown
## 관련 API

| 메서드 | 엔드포인트 | 설명 |
|--------|-----------|------|
| POST   | `/api/v1/detect` | YOLO 검출 |
| GET    | `/api/v1/results/{id}` | 결과 조회 |

## 관련 문서

- [시스템 개요](../system-overview/)
```

## 7. 문체

- **언어**: 한국어
- **문장**: 짧고 단정하게 (1문장 20자 이내 권장)
- **어조**: 운영/개발 문체 — 마케팅 문체 금지
- **근거**: 코드, 라우트, 컴포넌트 기준으로만 작성 (추측 금지)
- **금지어**: "좋습니다", "강력합니다", "멋집니다", "최고의", "혁신적인", "완벽한"
