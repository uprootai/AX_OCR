# /simplify - 코드 정리 커맨드

> 보리스 체니 전략 #8: Code Simplifier 서브 에이전트

## 목적

작업 완료 후 코드를 정리하고 단순화합니다.

## 체크리스트

### 1. 불필요한 코드 제거

- [ ] 사용하지 않는 import 제거
- [ ] 사용하지 않는 변수 제거
- [ ] 주석 처리된 코드 제거
- [ ] console.log / print 디버깅 코드 제거

### 2. 코드 단순화

- [ ] 중복 코드 통합
- [ ] 복잡한 조건문 단순화
- [ ] 불필요한 추상화 제거
- [ ] 과도한 에러 핸들링 제거

### 3. 파일 크기 검토

| 라인 수 | 상태 | 조치 |
|---------|------|------|
| < 500줄 | ✅ 양호 | 유지 |
| 500-1000줄 | ⚠️ 주의 | 리팩토링 고려 |
| > 1000줄 | ❌ 위반 | **즉시 분리** |

### 4. 네이밍 검토

- [ ] 의미 있는 변수명
- [ ] 일관된 네이밍 컨벤션
- [ ] 약어 최소화

## 실행 방법

```bash
# 특정 파일 정리
/simplify src/components/MyComponent.tsx

# 최근 변경 파일 정리
/simplify --changed
```

## 금지 사항

- 기능 추가 금지 (정리만)
- 새 파일 생성 금지
- 테스트 수정 금지
